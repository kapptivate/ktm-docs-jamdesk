This quickstart walks you through creating a test, running it on an agent, and reading the result. By the end you will understand the author → run → result loop that everything else in Kapptivate builds on.

<Steps>
  <Step>
    **Create a test.** Navigate to the **Tests** section in the sidebar for your Product. Click **Create**, select your channel (Web, Smartphone, or Cellular), and confirm. The test editor opens. Give your test a name and description, then confirm the action (device type and starting point: for example, a URL for web or an app bundle for mobile).
  </Step>
  <Step>
    **Add steps.** Inside the action, add steps to define what the test does. Each step is an action like clicking an element, typing text, navigating to a URL, waiting for content, or checking that something is on the page. For a web test, you might click a login button, type credentials, then check for a welcome message. Save the test.
  </Step>
  <Step>
    **Run the test.** Click the **Run** button on your test (in the test list) or inside the editor. Choose one or more agents to run against (for example, a Chrome browser for web, or a specific Smartphone device). The platform queues the test and starts execution. You can watch the run in real time from the **Executions** section.
  </Step>
  <Step>
    **View the result.** Once the test finishes, navigate to **Executions** in the sidebar. Find your test execution in the list and click into it. You will see:

    - **Status**: did the test pass, fail, or warn?
    - **Steps**: a breakdown of each action the test performed.
    - **Screenshots**: captures from each step, showing the state of the application.
    - **Timing**: how long each step took.
    - **Variables**: any input or output values used during the run.

    If a step failed, the screenshot and error message show you exactly where things went wrong.
  </Step>
</Steps>

## Next steps

Now that you have run a test, explore the platform further:

- **Organize tests**: use [collections](/tests/overview) to folder your tests by feature or team.
- **Parameterize tests**: add [input variables](/tests/create) so you can run the same test with different data.
- **Run on a schedule**: [create a monitor](/monitoring/create) to run your test continuously in production and alert you to failures.
- **Integrate with CI/CD**: use [API keys](/administration/api-keys) to trigger test runs from your pipeline.
