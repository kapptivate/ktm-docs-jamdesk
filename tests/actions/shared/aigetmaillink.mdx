---
title: AI get mail link
description: Extract an action link from email using AI and context intent.
---

Use AI to extract action links from emails based on an intent description. Specify what kind of link you're looking for (e.g., "password reset", "email verification"), and AI will identify the relevant link in the email. This is useful for testing password resets, email confirmations, and other email-driven flows.

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support mail link extraction equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| mailbox | Template | Yes | Email address or mailbox identifier to check. Supports templating (e.g., `${TEST_EMAIL}`). |
| intent | Template | Yes | Description of link purpose (e.g., "password reset", "email verification"). Helps AI identify the right link. |
| variable | String | No | Variable name to store extracted URL. If omitted, URL is only logged. |
| mocked_response | Object | No | Mock response for testing (JSON). Use in development/testing only. |

## Example

Extract a password reset link:

```json
{
  "type": "aigetmaillink",
  "version": "v1.0.0",
  "inputs": {
    "mailbox": "user@example.com",
    "intent": "password reset",
    "variable": "reset_link"
  }
}
```

Extract an email verification link:

```json
{
  "type": "aigetmaillink",
  "inputs": {
    "mailbox": "${VERIFICATION_EMAIL}",
    "intent": "email verification",
    "variable": "verify_url"
  }
}
```

## Notes

- AI uses the intent description to understand which link to extract.
- Extracted URL is stored in the variable for use in subsequent [navigate](/tests/actions/web/startingurl) actions.
- Requires mail service integration to be configured.
- Common uses: test password reset flows, verify account creation links, validate email-based onboarding.
