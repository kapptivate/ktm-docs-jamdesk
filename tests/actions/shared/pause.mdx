Pause test execution for an explicit wait period. This is useful when you need to wait for animations, backend processing, or other time-dependent events that can't be detected with element-based waits.

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support explicit pauses equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| duration | Template (ms) | Yes | Pause duration in milliseconds. Supports templating (e.g., `${WAIT_TIME}`). |

## Example

```json
{
  "type": "pause",
  "version": "v1.0.0",
  "inputs": {
    "duration": "2000"
  }
}
```

## Notes

- Pauses are absolute waits, not conditional. Use [waitfor](/tests/actions/shared/waitfor) for conditional waits.
- Duration can be dynamic via [variables](/concepts/core-concepts) (e.g., `${PAUSE_DURATION}`).
- Common uses: wait for CSS animations, allow backend processing, debug/inspect page state mid-test.
