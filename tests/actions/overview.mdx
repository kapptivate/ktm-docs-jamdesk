An action (also called a step) is a single operation in your test—a click, an assertion, a pause, or a variable assignment. Your test is simply an ordered list of actions that execute sequentially on the target channel. Each action receives inputs (a locator, a value, a timeout), executes on the device (browser or smartphone), and produces a result (pass/fail, extracted data, or a state change).

## How actions work

Every action follows the same lifecycle: it receives inputs (a selector, a value, a timeout), executes on your device, and produces a result. Many actions also support **step-level checks**—optional verifications like "the URL contains /success" or "the page loaded in under 2 seconds"—that run after the action completes. These checks can be marked as errors (fail the step) or warnings (log but continue).

Some actions support **templating**, which means you can use variables, globals, or template functions like `${NOW()}` or `${UUID()}` in values. You can also set test variables with one action and reference them in later steps.

The test framework supports both Web (desktop browsers, tablets) and Smartphone (Android, iOS) channels, but not all actions apply to both. For example, you can only `hover` on the Web, and you can only `tap` or `longpress` on smartphones.
