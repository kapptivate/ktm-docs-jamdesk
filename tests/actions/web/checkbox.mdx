---
title: Set checkbox
description: Legacy step that checks or unchecks a checkbox. Use Click instead.
---

<Warning>
  This step no longer appears in the step palette. Existing tests that use it keep running. For new tests, use [Click](/tests/actions/web/click) on the checkbox. Note that a click toggles the state instead of setting it, so verify the result with [Verify element](/tests/actions/web/verifyelement) when it matters.
</Warning>

Set a checkbox to a target state. The step only clicks when the current state differs from the requested one, so it is safe to repeat.

## Parameters

| Parameter | JSON field | Required | Description |
|-----------|------------|----------|-------------|
| Target element | `locator.description` | Yes, unless a manual selector is set | Natural-language description of the checkbox. |
| Manual selector | `locator.selector` | No | XPath or CSS selector, used when AI is disabled or as the saved selector. |
| Set as checked | `checked` | No | The target state. Default: off (unchecked). |
| AI usage | `ai_strategy` | No | `inherit` (default, follows the test strategy), `smart`, or `always`. |
| Ignore error on this step | `ignore_errors` | No | The run continues with the next step if this one fails. Default: off. |

## In the scenario JSON

The step lives in `step_groups[].steps` of the test scenario.

```json
{
  "id": "step-1",
  "type": "checkbox",
  "version": "v1.0.0",
  "disabled": false,
  "inputs": {
    "locator": {
      "selector": [{ "type": "value", "data": { "value": "" } }],
      "description": [{ "type": "value", "data": { "value": "The terms and conditions checkbox" } }]
    },
    "checked": true,
    "enable_ai": true,
    "ai_strategy": "inherit",
    "ignore_errors": false
  },
  "use_capabilities": false,
  "capabilities": {}
}
```

## Results and metrics

- **Timing**: start, end, and duration (in seconds) recorded in the step result.
- **Status**: success, warning, or failed, with the error message on failure.
- **Screenshot**: captured after the step runs.
