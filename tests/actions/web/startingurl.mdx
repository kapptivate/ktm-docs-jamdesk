title: Starting URL

Set the starting URL for your web test. This action must be the first step in any web test and blocks test execution until the page loads successfully.

## Channels

- ✅ Web (desktop/tablet browsers)
- ❌ Smartphone
- ❌ Cellular

Starting URL is web-only and acts as the test entry point.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| url | Template (string) | Yes | The URL to navigate to. Supports templating (e.g., `${BASE_URL}/login`). |
| enable_requests_interception | Boolean | No | Enable HTTP request mocking/interception for testing with mocked responses. |

## Example

```json
{
  "type": "starting-url",
  "version": "v1.0.0",
  "inputs": {
    "url": "https://example.com/login",
    "enable_requests_interception": false
  }
}
```

## Notes

- Starting URL is always Step 0 (index 0) in web tests and cannot be moved.
- The test blocks until the page is fully loaded.
- URLs support [templating](/tests/configurations) for dynamic test environments (e.g., staging, production).
- Request interception allows you to mock API responses for isolated testing.
- Step-level checks (url, loading_time, step_duration) are available after navigation completes.
