---
title: FTP Download
description: Download a file over FTP and measure throughput.
---

Download a file from an FTP server over the SIM's data connection and measure throughput. Useful when your benchmark infrastructure is FTP-based or to compare protocols on the same network.

## Parameters

| Parameter | JSON field | Required | Description |
|-----------|------------|----------|-------------|
| Download link | `command.download_link` | Yes | FTP URL of the file. |
| Download size limit | `command.download_size_limit` | No | Stop after this many bytes. |
| FTP credentials | `command.ftp_username`, `command.ftp_password` | No | Login for the FTP server. |
| Download should fail | `expect_failure` | No | Invert the verdict. |

Plus the shared [connection settings](/tests/actions/cellular/overview#connection-settings) and DNS override (`command.override_dns`, `command.dns`).

## In the scenario JSON

```json
{
  "type": "DATA_FTP_DOWNLOAD",
  "owner": "SIM_1",
  "skip": false,
  "expect_failure": false,
  "command": {
    "download_link": "ftp://files.example.com/10MB.bin",
    "download_size_limit": "",
    "ftp_username": "test",
    "ftp_password": "secret",
    "access_technology": "",
    "default_apn": true,
    "keep_alive": false,
    "check_network": { "active": true },
    "override_dns": false,
    "dns": [],
    "options": { "artifacts": {} }
  },
  "assertions": [],
  "store": []
}
```

## Results and metrics

In the action report (`result`): `throughput_in_kbps`, `md5sum` (file integrity), `dns_server`, and `duration` in seconds.

In Analytics, each run feeds the FTP download measurement with `throughput_kbps` and `duration_seconds`, tagged by status, URL, APN, antenna, and access technology.
