Cellular actions are V1 TestingHouse operations for testing telecom protocols and real SIM behavior. These actions are distinct from web and smartphone actions; USSD and SMS testing occurs over cellular networks, not through web browsers or app UIs.

USSD (Unstructured Supplementary Service Data) and SMS are the primary protocols tested in cellular flows. Tests chain actions together to simulate real user interactions—sending USSD menu codes, waiting for responses, and verifying SMS delivery.

## Available Actions

| Action | Purpose |
|--------|---------|
| [USSD](/tests/actions/cellular/ussd) | Send a USSD menu code and wait for the network response. |
| [SMS (receive)](/tests/actions/cellular/sms) | Wait for and verify an incoming SMS message. |
| [USSD push](/tests/actions/cellular/ussd-push) | Wait for an unsolicited USSD message from the network. |

## Expected Results and Assertions

Each action supports assertions to verify the response. Define expected results in the test editor and then validate them in result review:

- **USSD:** Assert on response message, session status, duration, access technology, or extract variables.
- **SMS:** Assert on sender phone number, message content, concatenation count, or extract OTP codes.
- **USSD push:** Assert on session status or extract message values.

### Bulk Updating Expected Results

When reviewing test results, you can fix incorrect expected messages across multiple tests:

- [Update USSD Expected Result](/tests/cellular) — Bulk-fix USSD response messages that were incorrect.
- [Update SMS Expected Result](/tests/cellular) — Bulk-fix SMS message expectations.
- [Update USSD Push Expected Result](/tests/cellular) — Bulk-fix USSD push message expectations.

These bulk operations find all tests using the same expected message and update them at once, saving time during result review.
