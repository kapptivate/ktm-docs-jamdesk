---
title: FTP Upload
description: Upload a generated file over FTP and measure uplink throughput.
---

Upload a file of a chosen size to an FTP server over the SIM's data connection and measure uplink throughput.

## Parameters

| Parameter | JSON field | Required | Description |
|-----------|------------|----------|-------------|
| Upload link | `command.upload_link` | Yes | FTP URL to upload to. |
| File size | `command.file_size_in_byte` | Yes | Size of the generated payload, in bytes. |
| FTP credentials | `command.ftp_username`, `command.ftp_password` | No | Login for the FTP server. |
| Upload should fail | `expect_failure` | No | Invert the verdict. |

Plus the shared [connection settings](/tests/actions/cellular/overview#connection-settings) and DNS override (`command.override_dns`, `command.dns`).

## In the scenario JSON

```json
{
  "type": "DATA_FTP_UPLOAD",
  "owner": "SIM_1",
  "skip": false,
  "expect_failure": false,
  "command": {
    "upload_link": "ftp://files.example.com/incoming/",
    "file_size_in_byte": 5000000,
    "ftp_username": "test",
    "ftp_password": "secret",
    "access_technology": "",
    "default_apn": true,
    "keep_alive": false,
    "check_network": { "active": true },
    "override_dns": false,
    "dns": [],
    "options": { "artifacts": {} }
  },
  "assertions": [],
  "store": []
}
```

## Results and metrics

In the action report (`result`): `throughput_in_kbps`, `md5sum`, `dns_server`, and `duration` in seconds.

In Analytics, each run feeds the FTP upload measurement with `throughput_kbps` and `duration_seconds`, tagged by status, URL, APN, antenna, and access technology.
