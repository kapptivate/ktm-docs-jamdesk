---
title: Wait USSD Push
description: Wait for an unsolicited USSD message pushed by the network.
---

Wait for a network-initiated USSD push: a balance alert, a subscription notification, a promotional message. The action sends nothing; it watches the SIM and verifies the message when it arrives.

## Parameters

| Parameter | JSON field | Required | Description |
|-----------|------------|----------|-------------|
| Expected Message | `expected_result.message` | No | The push content you expect. The result records the match ratio. |
| Check reception from a previous step | `command.wait_from` | No | Start watching from a previous action's position instead of this one, for pushes triggered earlier in the flow. |
| Reception is optional | `optional_reception` | No | The test passes even if no push arrives; variables get their default values. Default: off. |

## In the scenario JSON

```json
{
  "type": "USSD_PUSH",
  "owner": "SIM_1",
  "skip": false,
  "command": {
    "wait_from": null,
    "options": { "artifacts": {} }
  },
  "expected_result": {
    "message": "Your balance is below 1.00"
  },
  "assertions": [],
  "store": []
}
```

## Results and metrics

In the action report (`result`):

| Field | Description |
|-------|-------------|
| `message` | The push content received. |
| `session` | Session state after the push: `alive` or `close`. |
| `ratio` | Match ratio (0 to 100) against the expected message. |
| `relative_duration` | How long after the watch started the push arrived, in seconds. |
| `dynamic_variables` | Values extracted from the push into variables. |

In Analytics, each run feeds the USSD push measurement with `reception_duration_seconds` and `duration_seconds`, tagged by status, antenna, and access technology.
