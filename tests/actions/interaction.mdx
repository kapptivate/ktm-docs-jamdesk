Interaction actions perform user gestures—clicks, taps, text input, scrolling, and keyboard commands. They apply to Web, Smartphone, or both depending on the action.

## Common parameters

Most interaction actions use these parameters:

| Parameter | Type | Notes |
|-----------|------|-------|
| `locator` | string | CSS selector, XPath, or AI-driven description (e.g., "the submit button"). |
| `wait_for` | object | Optional condition(s) to wait for before/after the action (clickable, displayed, enabled, exist). |
| `timeout` | number (ms) | Max time to wait for element to appear. Default is typically 5–10 seconds. |

## Web & smartphone interaction

### click

Click on an element.

| Parameter | Type | Required | Applies to |
|-----------|------|----------|-----------|
| `locator` | string | Yes | Web, Smartphone |
| `wait_for` | object | No | Web, Smartphone |
| `timeout` | number | No | Web, Smartphone |

**Step-level checks available**: `url`, `loading_time`, `step_duration`

Use CSS selectors (`.button-primary`), XPath (`//button[contains(text(), "Submit")]`), or natural language descriptions if AI-driven selection is enabled.

### doubleclick

Double-click on an element.

| Parameter | Type | Required | Applies to |
|-----------|------|----------|-----------|
| `locator` | string | Yes | Web, Smartphone |
| `wait_for` | object | No | Web, Smartphone |

Typically used for text selection or triggering double-click event handlers.

### input (type text)

Type text into an input field or textarea.

| Parameter | Type | Required | Applies to |
|-----------|------|----------|-----------|
| `locator` | string | Yes | Web, Smartphone |
| `value` | string (templated) | Yes | Web, Smartphone |
| `wait_for` | object | No | Web, Smartphone |

**Step-level checks available**: `step_duration`

The `value` parameter supports templating, so you can use test variables like `${email}` or template functions like `${NOW()}`. The field is cleared before new text is entered.

### select

Select an option from a dropdown or select list.

| Parameter | Type | Required | Notes |
|-----------|------|----------|-------|
| `locator` | string | Yes | CSS/XPath selector for the `<select>` element. |
| `select_by` | string | Yes | Selection mode: `visible_text`, `index`, or `attribute`. |
| `value` | string (templated) | Yes | The value to match (text, index, or attribute value). |
| `wait_for` | object | No | Optional post-select condition. |

**Applies to**: Web, Smartphone

**Step-level checks available**: `step_duration`

Examples:
- `select_by: "visible_text"`, `value: "Choose Option A"` → selects the option with visible text "Choose Option A"
- `select_by: "index"`, `value: "2"` → selects the option at index 2
- `select_by: "attribute"`, `value: "opt1"` → selects the option with the matching attribute (e.g., `data-value="opt1"`)

### checkbox

Check or uncheck a checkbox.

| Parameter | Type | Required | Applies to |
|-----------|------|----------|-----------|
| `locator` | string | Yes | Web, Smartphone |
| `checked` | boolean | Yes | true = check, false = uncheck. |
| `wait_for` | object | No | Web, Smartphone |

**Applies to**: Web, Smartphone

The action is idempotent: it sets the checkbox to the desired state regardless of its current state.

### keys (pressKeys)

Press keyboard keys, including modifiers and special keys.

| Parameter | Type | Required | Applies to |
|-----------|------|----------|-----------|
| `keys` | array of strings | Yes | Web, Smartphone |

**Applies to**: Web, Smartphone

**Step-level checks available**: (none documented)

Key names include:
- Standard: `Enter`, `Tab`, `Escape`, `Backspace`, `Space`
- Navigation: `ArrowUp`, `ArrowDown`, `ArrowLeft`, `ArrowRight`
- Modifiers: `Shift`, `Control`, `Alt`, `Meta` (Cmd on Mac)
- Chords: `Control+a`, `Meta+c`, `Shift+Tab`

Examples:
- `["Enter"]` → submit a form
- `["Control+a", "Delete"]` → select all and delete
- `["Escape"]` → close a modal

### scroll

Scroll a page or scrollable element.

| Parameter | Type | Required | Notes |
|-----------|------|----------|-------|
| `locator` | string | No | CSS/XPath selector for the scrollable element; if omitted, scrolls the window. |
| `behavior` | string | No | `smooth` or `auto` (instant). |
| `block` | string | No | Vertical alignment: `start`, `center`, `end`. |
| `inline` | string | No | Horizontal alignment: `start`, `center`, `end`. |

**Applies to**: Web, Smartphone

Uses `element.scrollIntoView()` with options to scroll an element into view or to scroll by offset.

## Web-only interaction

### rightclick

Right-click to trigger a context menu.

| Parameter | Type | Required | Applies to |
|-----------|------|----------|-----------|
| `locator` | string | Yes | Web only |
| `wait_for` | object | No | Web only |

Emits a `contextmenu` event. Verify the context menu state with a separate assertion.

### hover

Move the mouse over an element to trigger hover states.

| Parameter | Type | Required | Applies to |
|-----------|------|----------|-----------|
| `locator` | string | Yes | Web only |
| `wait_for` | object | No | Web only |

Emits a `mouseover` event and triggers CSS `:hover` pseudo-classes. Not applicable to smartphones.

### uploadfile

Upload a file via an HTML file input.

| Parameter | Type | Required | Applies to |
|-----------|------|----------|-----------|
| `locator` | string | Yes | Web only |
| `url` | string (templated) | Yes | URL to the file resource. |
| `wait_for` | object | No | Web only |

**Applies to**: Web only (desktop browsers)

The URL can point to a Kapptivate-hosted file or an external resource.

## Smartphone-only interaction

### tap

Tap an element on a smartphone.

| Parameter | Type | Required | Notes |
|-----------|------|----------|-------|
| `locator` | string | Yes | Selector for the tap target. |
| `approach` | number | No | 0 = selector-based (default), 1 = coordinate-based. |
| `coordinates` | object | No | If approach=1, `{ x: value, y: value }` pixel coordinates (templated). |
| `wait_for` | object | No | Optional post-tap condition. |

**Applies to**: Smartphone only

**Step-level checks available**: `step_duration`

### longpress

Long-press (hold touch) on an element.

| Parameter | Type | Required | Notes |
|-----------|------|----------|-------|
| `locator` | string | Yes | Selector for the touch target. |
| `duration` | number (ms, templated) | No | Hold duration in milliseconds. |
| `wait_for` | object | No | Optional post-longpress condition. |

**Applies to**: Smartphone only

**Step-level checks available**: `step_duration`

May trigger long-press menus or context actions. Duration can be templated to make it dynamic.

### hidekeyboard

Hide the virtual keyboard on a smartphone.

| Parameter | Type | Required | Applies to |
|-----------|------|----------|-----------|
| (none) | — | — | Smartphone only |

**Applies to**: Smartphone only

**Step-level checks available**: (none)

A no-op if the keyboard is already hidden.
