Show the currently logged-in user (from the stored OAuth token)

## Usage

```bash
ktm whoami [flags]
```

## Flags

| Flag | Type | Default | Description |
| --- | --- | --- | --- |
| `--creds-file` | string | — | Path to the fallback credentials file |
| `--no-keychain` | boolean | — | Operate on the file fallback instead of the OS keychain |

## Details

Display the identity carried by the stored OAuth token: subject (user ID),
email, granted scopes, issuer, and time until expiry.

This is a local-only read; it does NOT call the Authorization Server.
Information shown is read directly from the JWT payload (unverified
display only).

</Note>
