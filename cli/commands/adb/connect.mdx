Open an ADB tunnel to a remote phone

## Usage

```bash
ktm adb connect <phone-serial> [flags]
```

## Examples

```bash
# Connect to a phone (auto-registers with local ADB server)
ktm adb connect RF8NB07VDGA

# Custom port
ktm adb connect RF8NB07VDGA --port 7037
```

## Arguments

| Argument | Required |
| --- | --- |
| `phone-serial` | Yes |

## Flags

| Flag | Type | Default | Description |
| --- | --- | --- | --- |
| `--port` | int | `6037` | Local port to listen on |

## Details

Starts a local ADB tunnel to a specific phone on a remote smartphone robot.

A thin TCP relay bridges your local ADB server to the remote phone over a
secure WebSocket tunnel. The device auto-registers with your local ADB server,
so Android Studio picks it up automatically. Full support for port forwarding,
reverse forwarding, and JDWP debugging — your real local ADB server handles
all protocol concerns.
