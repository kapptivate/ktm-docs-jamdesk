List incidents (triggered alerts) on the platform. Incidents are created when a platform alert definition fires. They track the lifecycle from OPEN to CLOSED.

NOT the same as evaluate_alert which performs local on-demand analytics. To manage platform alerts, use list_alert_definitions.
