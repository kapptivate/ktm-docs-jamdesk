## Parameters

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `aggregation` | string | Yes | Time aggregation bucket (e.g. '1h', '5m') |
| `aggregations` | string | No | Comma-separated aggregation windows for peak usage (e.g. 'p95,max'). When provided, returns P95 peak usage across these windows instead of the standard time-series. |
| `detail` | boolean | No | Return null-elided JSON instead of summary (default: false) |
| `end` | string | Yes | End time in ISO 8601 format (e.g. '2024-01-02T00:00:00Z') |
| `operator` | string | Yes | Operator slug (= InfluxDB database name) |
| `resources` | string | Yes | Comma-separated device IDs (SIM: national_msisdn, web-agent: hostname, smartphone: UUID) |
| `start` | string | Yes | Start time in ISO 8601 format (e.g. '2024-01-01T00:00:00Z') |

## Description

The full description the agent receives for this tool:

```text
Get device resource usage over time, broken down by a configurable aggregation window. Returns usage percentage per resource. Use list_resources to find device IDs (SIM: national_msisdn, web-agent: hostname, smartphone: UUID). Response keyed by resource name. Maximum recommended range: 90 days. Pass aggregations (e.g. 'p95,max') to include peak usage data alongside the default aggregation. Use instead of query_metrics for device utilization %; no InfluxQL needed.
```
