Security policies define password requirements, two-factor authentication, and login rules for your workspace. Use them to enforce consistent security standards across teams and assign different policies to different groups of users.

<Info>
Only workspace administrators can create and edit custom security policies.
</Info>

## What you can do

- [**View and create policies**](/administration/security-policies/create): browse existing policies and create custom ones for your workspace.
- [**Configure password rules**](/administration/security-policies/password): set password requirements, history tracking, and expiration rules.
- [**Configure authentication**](/administration/security-policies/authentication): require two-factor authentication and define login policies.
- [**Manage policies**](/administration/security-policies/manage): assign users to policies, clone policies, and set a workspace default.

Users inherit their security policy at the workspace level. To manage user accounts and their policy assignments, see [users](/administration/users). To audit changes to policies and other workspace events, see the [event log](/administration/event-log).
