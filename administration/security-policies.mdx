Security policies define password requirements, two-factor authentication, and login rules for your workspace. Use them to enforce consistent security standards across teams and assign different policies to different groups of users.

<Info>
Only workspace administrators can create and edit custom security policies.
</Info>

## View security policies

Go to **Administration** > **Security policies** to see all policies in your workspace. The table shows built-in default policies and any custom policies you've created.

Each policy shows its name, type (custom or default environment), and status.

## Create a custom policy

1. Click **Create Policy** (top right)
2. Fill in the **Policy Name**
3. Configure each security requirement in the collapsible panels below
4. Click **Save**

## Configure password requirements

Expand the **Password Policy** panel to set rules for how users create passwords.

**Length**: Set minimum and maximum character length. You can leave either bound empty if you don't want a limit.

**Character composition**: Require a minimum number of lowercase letters, uppercase letters, numbers, and special characters. For example, require at least 1 uppercase letter and 1 number.

**Consecutive characters**: Limit the number of times the same character can appear in a row (e.g., no "aaa").

**Username rules**:

- Toggle **Forbid username equality** to prevent users from setting a password identical to their username
- Toggle **Forbid username containing** to prevent passwords that include their username as a substring
- Toggle **Forbid whitespaces** to disallow spaces in passwords

## Configure password history

Expand the **Password History** panel and toggle **Remember previous passwords** to prevent users from reusing old passwords. When enabled, Kapptivate tracks previous passwords and rejects reuse.

## Configure password lifetime

Expand the **Password Life** panel to set when and how often users must change their password.

**Change after reset**: Force users to change their password the next time they log in after an admin resets it.

**Change after first login**: Force users to change their password on their first login.

**Limit password lifetime**: Require users to change their password after a set number of days. When enabled, specify the expiration period.

**Lifetime reminder**: Send users a reminder email before their password expires. Configure:

- **Trigger duration**: When to start sending reminders (e.g., 7 days before expiration)
- **Interval**: How often to repeat the reminder

## Configure two-factor authentication

Expand the **Two Factor Authentication** panel to require 2FA for users assigned to this policy.

## Configure login policies

Expand the **Login Policy** panel to set rules for how users can log in — such as allowed login methods and IP restrictions.

## Assign users to a policy

Expand the **Users** panel to see which users this policy applies to. Search by name, username, or email to find users.

To add users to the policy, click **Add Users**, select them from the modal, and confirm. To remove a user, click the delete button next to their name.

## Clone a policy

Cloning copies a policy as a starting point for a new one. Open the policy you want to clone, click **Clone**, then modify the copy as needed. The cloned policy is created as a custom (non-default) policy.

## Set a default policy

Mark a policy as the default for your workspace. New users are automatically assigned the default policy unless you assign them to a different one.

Only one policy can be the default at a time.
