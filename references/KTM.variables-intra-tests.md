## 1. Set variable step

**Source:** both

**Docs coverage:** outdated — The documentation only describes configuring the step via JSON parameters and does not reflect the current visual UI for adding and configuring the 'Set variable' step directly in the test builder.

Users can add a 'Set variable' step to their test to create a variable that lives only within the scope of that specific test. This is done by selecting the step from the 'Add step' menu, providing a value, and giving the variable a name.

**Timestamps:** 00:19

## 2. Use variable menu

**Source:** visual

**Docs coverage:** new — The documentation does not mention the visual `{}` menu used to insert variables into input fields.

When configuring a step (like a 'Click' step with text input), users can click the `{}` icon or select 'Use variable' within the input field to open a variables menu. This menu allows users to insert variables instead of static text and is divided into three tabs: 'Previous steps', 'Random', and 'Variables'.

**Timestamps:** 00:26

## 3. Previous steps variables

**Source:** both

**Docs coverage:** new — The documentation fails to mention that variables from previous steps are automatically tracked and can be selected visually from a list.

In the 'Use variable' menu, the 'Previous steps' tab lists all variables generated by preceding steps in the test. It shows the variable name alongside the step number where it was created (e.g., 'SmsCode - Step 1', 'MailCode - Step 2', 'Variable - Step 4').

**Timestamps:** 00:31

## 4. Random data generators

**Source:** visual

**Docs coverage:** new — The documentation mentions `${RANDOM()}` as a template function but does not document the visual UI presets for generating names, cities, and addresses.

The 'Random' tab in the variables menu provides presets to generate dynamic fake data during the test run. Available preset options include 'First name', 'Last name', 'City', and 'Street address', as well as a 'Custom' option.

**Timestamps:** 00:39

## 5. Custom random format

**Source:** visual

**Docs coverage:** new — The documentation completely omits the custom random format builder and its specific configuration options for text, numbers, and dates.

Selecting 'Custom' from the Random tab opens a dialog to configure specific random data formats. Users can choose a Type ('Text', 'Number', 'Date'). For 'Text', users can toggle character sets (a-z, A-Z, 0-9, special characters) and set a specific character length. For 'Date', users can select from various formats such as 'DD/MM/YYYY', 'MM/DD/YYYY', 'MM/YYYY', 'DD/MM', and 'M, D Y'.

**Timestamps:** 00:43, 00:46

## 6. Global variables and groups

**Source:** both

**Docs coverage:** new — The documentation mentions referencing global variables via templating but does not cover the UI for selecting them or the concept of variable groups.

The 'Variables' tab in the 'Use variable' menu displays global variables and variable groups configured outside the test scope. Users can select standalone global variables (e.g., 'Firstname', 'MDPKyota') or expand variable groups (like 'Default') to select specific variables contained within them (e.g., 'PASSWORD_KAPPTIVATE').

**Timestamps:** 00:58, 01:01

## 7. Environment variables overview

**Source:** visual

**Docs coverage:** new — The documentation does not mention the Environment panel where users can review active test and global variables.

In the right-hand panel under the 'Environment' tab, users can view a summary of all variables currently utilized in the test. This panel is split into 'Test variables' (listing variables created within the test) and 'Global variables' (listing any external or group variables referenced in the test, showing their values or indicating if they are masked).

**Timestamps:** 01:05, 01:21
