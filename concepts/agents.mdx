An agent is the on-premise hardware and software that runs your tests on real devices and SIMs. All agents are installed at your locations and managed by your team. Kapptivate supports five user-facing agent types, each designed for a specific test channel.

## Web agent (Ethernet)

A server or machine that runs browser-based tests from a specific network location.

**What it does:**
- Automates website and web application testing
- Launches browsers and executes test steps
- Supports multiple simultaneous tests

**Configuration:**
- Hostname or IP address
- Parallel test capacity (how many tests can run at once)
- Automatic reboot and update scheduling
- Network configuration and credentials for authentication

**Statuses:** Online, Offline, Rebooting, Maintenance, Waiting for Config (initial setup), or Test-only mode.

## Smartphone agent

A controller that manages real Android and iOS devices for mobile app testing.

**What it does:**
- Runs tests against native Android and iOS apps
- Captures device properties (manufacturer, model, SDK version, screen resolution)
- Can reset apps before each test

**Connected devices:** Real phones and tablets in your lab.

**Statuses:** Online, Offline, Rebooting, Maintenance, or Test-only mode.

## Cellular agent

A specialized device that holds multiple SIM cards and runs tests on real cellular networks.

**What it does:**
- Executes tests on physical SIM cards
- Handles USSD menu flows and SMS messaging
- Monitors network registration and signal strength
- Manages APN configurations for each SIM

**Connected devices:** SIM cards in numbered slots. Each slot can hold one SIM and stores its registration status, signal level, and network operator information.

**Statuses:** Online, Offline, Rebooting, Maintenance, or Test-only mode.

## Mono Agent (robot digital)

A single combined device that performs both Web and Smartphone testing.

**What it does:**
- Runs Web tests via an embedded browser
- Runs Smartphone tests via embedded Android/iOS simulators or connected devices
- Lets you test both web and mobile versions of an application with one agent

**Typical use case:** When you need to test both a website and its mobile app but prefer a single on-premise appliance.

**In the UI:** Mono agents are generally treated as smartphone agents for most operations.

## Passive agent

A lightweight monitoring endpoint that collects metrics and health data without running active tests.

**What it does:**
- Polls service health and collects metrics
- Monitors logs and system performance
- Requires no hostname or active test execution

**Common use case:** Continuous monitoring of backend services or network endpoints that don't require interactive test steps.

## Agent status overview

All agents report their operational status:

| Status | Meaning |
|--------|---------|
| **Online** | Agent is reachable and ready to run tests |
| **Offline** | Agent is unreachable or stopped |
| **Rebooting** | Agent is currently rebooting |
| **Maintenance** | Agent is in maintenance mode (manual or scheduled) |
| **Waiting for Config** | Agent has initialized but lacks required setup (Web agents only) |
| **Test-only** | Agent is reserved for test-only runs; cannot run production tests |
