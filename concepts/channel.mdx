A **Channel** is a broad category of endpoints you test: Web, Smartphone, or Cellular. Each channel has its own device types and agents. On top of these, **APIs** can be tested as well, on their own or alongside any channel.

- **Web**: test desktop and mobile browsers.
- **Smartphone**: test native iOS and Android apps.
- **Cellular**: test SMS, USSD, or cellular-dependent flows.
- **API**: test HTTP endpoints, either standalone or combined with another channel.

A simple test runs on a single channel, optionally paired with API, for example Web only, Web + API, Smartphone + API, or USSD (SIM) + API.

For more complex flows, advanced orchestration lets a single test span multiple devices: start a scenario on Web and finish it on a Smartphone, for instance.

For how each channel maps to an agent and the devices it controls, see [Web & API](/agents-devices/web-api), [Smartphone](/agents-devices/smartphone), and [SIM](/agents-devices/sim).
