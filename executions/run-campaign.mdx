A [campaign](/tests/suites) bundles tests that make sense together; running it executes them as one batch. Like a [manual run](/executions/run), a campaign run goes through a confirmation step where you adjust the batch before launching.

## The run confirmation

Click **Run** on a campaign (from the campaigns table or the editor). The **Run confirmation** page opens for a last review:

- **Name**: the execution name, defaulting to the campaign name; rename it to mark the occasion (a release check, a regression pass).
- **Your tests**: every test in the campaign, with its **owner** (the device or agent that will run it) and **input variables**, both adjustable for this run only.

Click **Confirm run** (top right) to launch.

## The run queue

Queued runs appear in the **Run queue** sidebar. **Clear queue** empties all pending runs after a confirmation.

![The Run queue sidebar with a queued campaign card and the Clear queue action](/images/campaign-run-queue.webp)

## Follow the execution

The campaign lands in the [executions list](/executions/overview) and opens as a [multi-test execution](/executions/results#multi-test-executions): summary counters, an execution time chart, and each test's detailed results one click away in the sidebar.

<Tip>
To launch campaigns from a script or a pipeline instead of the UI, see [Run campaigns from the CLI](/cli/guides/run-campaigns).
</Tip>
