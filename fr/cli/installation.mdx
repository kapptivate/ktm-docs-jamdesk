`ktm` est distribué sous la forme d'un binaire unique et autonome pour Linux, macOS et Windows. Une fois placé sur votre PATH, vous vous authentifiez avec votre clé API Kapptivate et vous êtes prêt.

## Installer ktm

Exécutez la commande correspondant à votre plateforme :

<Tabs>
  <Tab title="macOS / Linux">

```bash
curl -fsSL https://artefacts.kapptivate.com/ktm/install.sh | bash
```

  </Tab>
  <Tab title="Windows">

```powershell
irm https://artefacts.kapptivate.com/ktm/install.ps1 | iex
```

  </Tab>
</Tabs>

Le script télécharge la dernière version stable de `ktm`, vérifie son checksum et l'ajoute à votre PATH. Définissez `KAPPTIVATE_VERSION=1.4.0` pour fixer une version spécifique.

Vérifiez l'installation :

```bash
ktm --version
```

## Authentification

`ktm` lit votre clé depuis la variable d'environnement `KAPPTIVATE_API_KEY` :

```bash
export KAPPTIVATE_API_KEY="your-key"
ktm status
```

`ktm status` vérifie la connectivité, l'authentification et la version du CLI en un seul appel. Un résultat propre signifie que vous êtes prêt.

<Tip>
  En CI, stockez la clé comme secret et exposez-la en tant que `KAPPTIVATE_API_KEY`. Ne la codez jamais en dur dans un fichier de workflow.
</Tip>
