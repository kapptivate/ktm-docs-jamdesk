Récupérer une seule collection avec son chemin de navigation

## Utilisation

```bash
ktm collections get <uid> [flags]
```

## Exemples

```bash
ktm collections get --operator my-op 8294de2a-ecd9-4202-9169-7d55d1328e22
ktm collections get --operator my-op <uid> -o json | jq '.path'
```

## Arguments

| Argument | Requis |
| --- | --- |
| `uid` | Oui |

## Flags

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--operator` | string | — | Slug de l'opérateur (requis) |

## Détails

Retourne les métadonnées de la collection (name, parent_uid, org_id, trash_uid)
ainsi que son chemin de navigation depuis la racine. Utile avant
une mutation pour confirmer la portée, ou pour découvrir l'UID de la corbeille pour les flux de nettoyage.
