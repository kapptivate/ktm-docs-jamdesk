Déplacer une collection vers la corbeille de l'organisation (suppression douce)

## Utilisation

```bash
ktm collections delete <uid> [flags]
```

## Exemples

```bash
ktm collections delete --operator my-op <uid>              # refuses without --yes
ktm collections delete --operator my-op --yes <uid>        # moves to trash
```

## Arguments

| Argument | Requis |
| --- | --- |
| `uid` | Oui |

## Flags

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--operator` | string | — | Slug de l'opérateur (requis) |
| `--yes` | boolean | — | Confirmer la suppression irréversible |

## Détails

Déplace la collection vers la corbeille de l'organisation. La collection
et ses éléments sont récupérables jusqu'à ce que la corbeille soit vidée (action admin séparée).
Malgré le nom "delete", ce n'est PAS une suppression définitive.

La commande refuse l'opération sans --yes car les agents doivent confirmer la portée
avant de déplacer des collections.
