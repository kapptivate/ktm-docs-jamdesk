Valider un fichier JSON de test sans le créer

## Utilisation

```bash
ktm tests validate [flags]
```

## Exemples

```bash
# Valider un fichier de test
ktm tests validate --from-file test.json

# Pipeline : générer, valider, créer
ktm tests template ETH_HTTP_API > test.json
# ... modifier test.json ...
ktm tests validate --from-file test.json
ktm tests create --from-file test.json --product my-operator~my-product
```

## Flags

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--from-file` | string | — | Chemin vers le fichier JSON à valider (requis) |

## Détails

Validation dry-run d'un payload JSON de test sans appeler l'API.

Vérifie : les types d'action, la présence d'un owner, les champs de commande requis, la structure des variables.
Retourne des erreurs et avertissements structurés avec des chemins JSON.
