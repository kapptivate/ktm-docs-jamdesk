Définition complète du test avec actions, dépendances et configuration

## Utilisation

```bash
ktm tests inspect <id> [flags]
```

## Exemples

```bash
ktm tests inspect 99
ktm tests inspect 99 -o json | jq '.actions[0]'
```

## Arguments

| Argument | Requis |
| --- | --- |
| `id` | Oui |

## Détails

Récupérer la forme complète du test, incluant les actions, les variables et la configuration.
C'est le payload complet de la définition de test, utile pour comprendre exactement ce qu'un test
fait avant de le modifier ou de le cloner.

Pour un résumé plus léger (sans les actions), utilisez 'ktm tests get'.
