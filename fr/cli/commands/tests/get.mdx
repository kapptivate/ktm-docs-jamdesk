Obtenir les détails d'un test incluant les définitions de variables et les slots owner

## Utilisation

```bash
ktm tests get <id> [flags]
```

## Exemples

```bash
ktm tests get 99
ktm tests get 99 -o json | jq '{name, id, variables}'
```

## Arguments

| Argument | Requis |
| --- | --- |
| `id` | Oui |

## Détails

Récupérer les détails d'un test : métadonnées, variables (input_variables, output_variables,
owners) et informations de collection. Cela retourne les définitions de variables, pas
la liste complète des actions. Pour la définition complète incluant les actions, utilisez
'ktm tests inspect'.
