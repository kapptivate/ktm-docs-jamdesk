Supprimer un ou plusieurs tests (déplacement vers la corbeille ; récupérable via 'tests undelete')

## Utilisation

```bash
ktm tests delete <id>... [flags]
```

## Exemples

```bash
ktm tests delete 99 --yes
ktm tests delete 99 100 101 --yes
ktm tests delete 117 --kind 2 --yes           # reusable
```

## Arguments

| Argument | Requis |
| --- | --- |
| `id` | Oui |

## Flags

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--kind` | int | `1` | Kind du test : 1=test (défaut), 2=reusable |
| `--yes` | boolean | — | Confirmer la suppression |

## Détails

Envoie les tests dans la collection test-trash du produit. Malgré le nom "delete",
cette opération est réversible : exécutez 'ktm tests undelete &lt;id&gt;' pour récupérer un test
depuis la corbeille. Le service collections prend en charge une action admin 'empty trash'
qui purge la corbeille de manière permanente ; cela est hors du périmètre de ce CLI.

--kind contrôle dans quelle table de kind de test le backend cherche :
  1  test standard (défaut)
  2  test reusable

Refuse sans --yes.

## Notes

- Suppression vers la corbeille, récupérable avec [`ktm tests undelete`](/fr/cli/commands/tests/undelete). Pour simplement masquer un test des listes, utilisez [`ktm tests archive`](/fr/cli/commands/tests/archive).
