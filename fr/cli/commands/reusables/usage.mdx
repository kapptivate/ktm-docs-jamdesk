Lister les tests qui intègrent ce reusable

## Utilisation

```bash
ktm reusables usage <id> [flags]
```

## Exemples

```bash
ktm reusables usage 117
ktm reusables usage 117 --limit 200 -o json | jq '.entries'
```

## Arguments

| Argument | Requis |
| --- | --- |
| `id` | Oui |

## Flags

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--limit` | int | — | Nombre maximum d'entrées à développer (défaut : 50, max : 200) |

## Détails

Lit le tableau reused_by du reusable (un tableau d'identifiants de tests), puis
développe chaque identifiant en \{id, name\} via un GET par identifiant. Limité à 50 entrées par
défaut (max 200) pour borner la latence. Rejette si &lt;id&gt; est un test standard ; utilisez
'ktm reusables in-test' pour la direction inverse.
