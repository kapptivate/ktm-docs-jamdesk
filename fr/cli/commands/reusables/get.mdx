Obtenir la définition complète d'un reusable (rejette les identifiants de tests non-reusable)

## Utilisation

```bash
ktm reusables get <id> [flags]
```

## Exemples

```bash
ktm reusables get 117
ktm reusables get 117 -o json | jq '{id, name, kind, reused_by}'
```

## Arguments

| Argument | Requis |
| --- | --- |
| `id` | Oui |

## Détails

Récupère la définition complète du test pour un reusable (kind=2). Si l'identifiant
pointe vers un test standard (kind=1), la commande renvoie une erreur et suggère
d'utiliser 'ktm tests inspect' à la place. Utilisez cette commande pour consulter les actions, variables et
reusables intégrés avant de les modifier.
