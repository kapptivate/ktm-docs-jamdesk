Lister les reusables intégrés dans un test standard

## Utilisation

```bash
ktm reusables in-test <test_id> [flags]
```

## Exemples

```bash
ktm reusables in-test 204
ktm reusables in-test 204 --limit 100
```

## Arguments

| Argument | Requis |
| --- | --- |
| `test_id` | Oui |

## Flags

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--limit` | int | — | Nombre maximum d'entrées à développer (défaut : 50, max : 200) |

## Détails

Inverse de 'usage'. Étant donné un identifiant de test standard (kind=1), retourne les
reusables intégrés via des actions 'import' (le tableau reused_tests de la définition
complète du test), développés en \{id, name\}. Rejette si &lt;test_id&gt; pointe vers un
reusable lui-même.
