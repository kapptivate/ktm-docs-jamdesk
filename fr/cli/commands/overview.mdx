## Flags globaux

Ces flags s'appliquent à toutes les commandes :

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--api-key-file` | string | — | Chemin vers le fichier contenant la clé API (alternative à la variable d'environnement KAPPTIVATE_API_KEY) |
| `-d, --debug` | boolean | — | Active la sortie de débogage : requêtes HTTP, timing et erreurs détaillées sur stderr |
| `--host` | string | — | Hôte API (env : KAPPTIVATE_HOST, défaut : app.kapptivate.com) |
| `--no-color` | boolean | — | Désactive la sortie colorée (aussi : variable NO_COLOR, TERM=dumb) |
| `--no-proxy` | boolean | — | Désactive le proxy HTTP de l'environnement |
| `-o, --output` | string | `json` | Format de sortie : json, table ou plain (TSV) |
| `-q, --quiet` | boolean | — | Supprime les lignes de log de niveau info sur stderr, affiche uniquement les erreurs |
| `--skip-tls` | boolean | — | Ignore la vérification des certificats TLS (non sécurisé) |
