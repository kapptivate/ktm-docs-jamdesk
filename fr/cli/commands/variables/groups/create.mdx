Créer un groupe de variables avec des variants

## Utilisation

```bash
ktm variables groups create [flags]
```

## Exemples

```bash
ktm variables groups create --product acme~web --name "Envs" --variants "Dev,Staging,Prod"
ktm variables groups create --product acme~web --name "Regions" --variants "US,EU,APAC" -o json
```

## Flags

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--name` | string | — | Nom du groupe (requis) |
| `--product` | string | — | Slug du produit (requis) |
| `--variants` | string | — | Noms de variants séparés par des virgules (requis) |

## Détails

Créer un nouveau groupe de variables sous un produit. --variants est une liste séparée par des virgules
de noms de variants (ex. "Dev,Staging,Prod"). Le PREMIER variant devient le
défaut selon la convention de la plateforme. Après la création, gérez les variants via
'ktm variables variants &lt;verb&gt;'.
