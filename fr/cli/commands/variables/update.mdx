Mettre à jour le nom, la description et/ou la default_value d'une variable (propriétés uniquement)

## Utilisation

```bash
ktm variables update <id> [flags]
```

## Exemples

```bash
ktm variables update 17 --description "DB hostname"
ktm variables update 17 --default-value "new-default"
ktm variables update 17 --name DB_HOST_V2
```

## Arguments

| Argument | Requis |
| --- | --- |
| `id` | Oui |

## Flags

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--default-value` | string | — | Nouvelle valeur par défaut |
| `--description` | string | — | Nouvelle description (optionnelle, recommandée) : écrase la valeur actuelle ; utile pour clarifier l'objectif |
| `--name` | string | — | Nouveau nom |

## Détails

Propriétés uniquement : cette commande ne touche PAS les valeurs par variant. Pour changer la
valeur d'un variant, utilisez 'ktm variables set-value' ; pour copier des valeurs entre variants,
utilisez 'ktm variables variants copy-values'. Le type de variable est immuable sur la
plateforme ; cette commande le conserve.

## Notes

- Met à jour les propriétés d'une variable (nom, description, valeur par défaut). Pour définir la valeur d'un seul variant, utilisez [`ktm variables set-value`](/fr/cli/commands/variables/set-value).
