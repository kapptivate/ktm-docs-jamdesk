Définir la valeur d'UN variant sur une variable groupée (conserve les autres variants)

## Utilisation

```bash
ktm variables set-value <id> [flags]
```

## Exemples

```bash
ktm variables set-value 17 --variant prod --value prod.db.example.com
ktm variables set-value 17 --variant 42 --value 0.0.0.0
```

## Arguments

| Argument | Requis |
| --- | --- |
| `id` | Oui |

## Flags

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--value` | string | — | Nouvelle valeur pour ce variant |
| `--variant` | string | — | Identifiant ou nom du variant (requis) |

## Détails

Modification atomique de la valeur par variant. --variant accepte un identifiant ou un nom de variant ; le
variant cible doit exister sur la variable. Erreur lorsqu'elle est appelée sur une variable
globale (sans variants) : utilisez 'ktm variables update --default-value' à la place.

## Notes

- Définit la valeur d'UN variant sur une variable groupée, en conservant les autres variants. Pour changer le nom, la description ou la valeur par défaut d'une variable, utilisez [`ktm variables update`](/fr/cli/commands/variables/update).
