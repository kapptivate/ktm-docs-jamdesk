Mise à jour partielle d'une Tests campaign (PATCH)

## Utilisation

```bash
ktm test-campaigns update <id> [flags]
```

## Exemples

```bash
ktm test-campaigns update 42 --name "Regression v2"
ktm test-campaigns update 42 --test-list @newlist.json
```

## Arguments

| Argument | Requis |
| --- | --- |
| `id` | Oui |

## Flags

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--description` | string | — | Nouvelle description |
| `--name` | string | — | Nouveau nom |
| `--test-list` | string | — | Fichier JSON (ou '-') remplaçant la test_list actuelle |

## Détails

Appliquer une mise à jour partielle. Toute combinaison de --name, --description ou
--test-list est autorisée. Si --test-list est fourni, il remplace entièrement la
test_list actuelle (les entrées existantes sont remplacées).
