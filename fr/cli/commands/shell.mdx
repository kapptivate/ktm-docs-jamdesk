## Détails

Envoyez des commandes directement aux appareils et consultez les résultats en temps réel.
Contrairement aux tests, les commandes shell n'ont pas d'assertions : elles envoient une commande et retournent le résultat brut.

Types d'appareils pris en charge :
  SIM (owner_kind=S) :      USSD, SMS, appels, opérations de données via réseau cellulaire
  Ethernet (owner_kind=E) : HTTP API, ping, DNS, transferts via web agents

Les smartphones ne sont pas pris en charge (architecture WebSocket uniquement).

Les sessions USSD sont avec état : composer un shortcode ouvre une session, et envoyer des numéros
d'option de menu (1, 2, 3) permet de naviguer plus profondément dans l'arborescence du menu. Les sessions expirent après
environ 30s d'inactivité. Utilisez --wait pour voir le texte du menu en ligne.

Certains flux déclenchent des événements asynchrones (ex. attente SMS après une sélection USSD ou un appel API).
Utilisez 'shell list --from-date &lt;timestamp&gt;' pour interroger les nouveaux shells depuis le début de votre session.

Note : les cartes SIM et les web agents sont des ressources partagées. Les monitors actifs exécutent des tests sur les mêmes
appareils selon un calendrier. Si un shell retourne Blocked ou Timeout de manière inattendue, il peut s'agir d'une
contention de ressources : attendez 15-30s et réessayez. Utilisez 'monitors toggle' pour mettre en pause les monitors si nécessaire.
