Obtenir l'utilisation des ressources d'appareils par intervalles d'une heure et de 5 minutes

## Utilisation

```bash
ktm metrics resource-usage [flags]
```

## Exemples

```bash
# Hourly usage for two SIMs over one day
ktm metrics resource-usage --operator acme --resources 0612345678,0698765432 --start 2024-01-01T00:00:00Z --end 2024-01-02T00:00:00Z

# 5-minute granularity
ktm metrics resource-usage --operator acme --resources 0612345678 --aggregation 5m --start 2024-01-01T00:00:00Z --end 2024-01-01T06:00:00Z
```

## Flags

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--aggregation` | string | `1h` | Fenêtre d'agrégation temporelle (ex. '1h', '5m') |
| `--end` | string | — | Heure de fin au format ISO 8601 (ex. '2024-01-02T00:00:00Z') |
| `--operator` | string | — | Slug de l'opérateur (= nom de base de données InfluxDB) |
| `--resources` | string | — | ID d'appareils séparés par des virgules |
| `--start` | string | — | Heure de début au format ISO 8601 (ex. '2024-01-01T00:00:00Z') |

## Détails

Obtenir l'utilisation des ressources d'appareils par intervalles de temps.

Retourne le pourcentage d'utilisation par ressource, ventilé par la fenêtre d'agrégation
spécifiée (défaut 1h). Utilisez cette commande pour analyser les patterns d'utilisation des appareils dans le temps.

Interroge le measurement "resource_usage" dans InfluxDB. Les ID d'appareils correspondent
aux ressources découvertes via 'ktm resources list' (SIM : national_msisdn,
web-agent : hostname, smartphone : UUID).

Plage temporelle maximale recommandée : 90 jours.
