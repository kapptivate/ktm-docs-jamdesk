Mettre à jour ktm vers la dernière version (ou une version spécifique)

<Note>Aliases: update, upgrade, self-update</Note>

## Utilisation

```bash
ktm update [flags]
```

## Exemples

```bash
# Update to the latest stable release
ktm update

# Only check whether an update is available (no install)
ktm update --check

# Follow the pre-release channel
ktm update --channel beta

# Pin / roll back to an exact version
ktm update --version 1.3.2 --yes

# Reinstall the current version
ktm update --force
```

## Flags

| Flag | Type | Défaut | Description |
| --- | --- | --- | --- |
| `--channel` | string | — | Canal de version : stable ou beta (défaut : marqueur d'installation, sinon stable) |
| `--check` | boolean | — | Indique si une mise à jour est disponible, sans l'installer |
| `--force` | boolean | — | Réinstalle même si déjà à jour |
| `--verify-signature` | boolean | — | Exige la vérification de la signature cosign de checksums.txt |
| `--version` | string | — | Installe une version exacte (ex. 1.3.2) ; permet le retour à une version antérieure |
| `-y, --yes` | boolean | — | Ignore l'invite de confirmation (auto-oui si pas un TTY) |

## Détails

Télécharge et installe une version plus récente de ktm depuis le serveur de versions Kapptivate.

Par défaut, met à jour vers la version la plus récente de votre canal d'installation :
  stable   versions de production uniquement (défaut)
  beta     inclut les pré-versions

Le binaire en cours d'exécution est vérifié (sha256, obligatoire) et remplacé de façon atomique.
Le canal et la source de téléchargement sont lus depuis le marqueur d'installation écrit par
l'installateur ; les flags les remplacent. Fixez ou revenez en arrière avec --version &lt;X&gt;.

Cette commande ne communique qu'avec le serveur de versions public ; elle ne nécessite pas
d'authentification.

</Note>
