---
title: Créer une métrique
description: Enregistrer une métrique de performance personnalisée pendant l'exécution d'un test.
---

Enregistrez des métriques de performance personnalisées pendant vos tests. Les métriques sont stockées au format InfluxDB et peuvent être utilisées pour le monitoring de performance, les alertes et l'analyse de tendances entre les exécutions.

## Canaux

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

Tous les canaux supportent l'enregistrement de métriques de la même manière.

## Paramètres

| Paramètre | Requis | Description |
|-----------|--------|-------------|
| Measurement | Oui | Nom de la métrique (ex. `form_submit_time`, `api_response_ms`). |
| Fields | Oui | Paires clé-valeur de mesures numériques ou booléennes (les valeurs mesurées). |
| Tags | Non | Paires clé-valeur pour le regroupement et le filtrage des métriques (ex. nom du test, type d'utilisateur, région). |

## Notes

- Les tags sont des dimensions optionnelles pour le filtrage et le regroupement des métriques dans les dashboards.
- Les métriques sont agrégées entre les exécutions pour l'analyse de tendances et les alertes.
- Usages courants : suivre des durées personnalisées, enregistrer des indicateurs de succès/échec, mesurer des résultats métier.
