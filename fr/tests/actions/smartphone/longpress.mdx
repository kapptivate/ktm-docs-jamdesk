Appuyez longuement sur un élément : ouvrir un menu contextuel sur un élément de liste, démarrer une poignée de glissement, ou déclencher tout geste d'appui long supporté par l'application.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Élément cible | Oui, sauf si un selector manuel est défini | Description en langage naturel de l'élément. L'IA l'utilise pour trouver l'élément. |
| Selector manuel | Non | Selector XPath dans l'arbre UI de l'application, utilisé lorsque l'IA est désactivée ou comme selector enregistré. |
| Utilisation de l'IA | Non | `smart` (par défaut), `always` ou `inherit`. |
| Activer l'IA | Non | Par défaut : désactivé. Activez-le pour localiser l'élément à partir de la description. |
| Ignorer l'erreur sur ce step | Non | L'exécution continue avec le step suivant si celui-ci échoue. Par défaut : désactivé. |

## Résultats et métriques

- **Timing** : début, fin et durée (en secondes) enregistrés dans le résultat du step.
- **Statut** : succès, avertissement ou échec, avec le message d'erreur en cas d'échec.
- **Capture d'écran** : prise après l'appui long, montrant généralement les actions contextuelles ouvertes.
- **Utilisation de l'IA** : si l'IA a localisé l'élément, le selector corrigé est enregistré sur le test et le coût IA est ajouté à l'exécution.
