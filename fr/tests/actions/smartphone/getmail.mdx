Vérifiez que votre application envoie bien un email : un message de bienvenue, une confirmation de commande, une facture. Le step attend que l'email arrive dans une boîte de test et échoue si rien n'arrive.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Boîte mail | Oui | L'adresse de la boîte de test qui doit recevoir l'email. |
| Ignorer l'erreur sur ce step | Non | L'exécution continue avec le step suivant si aucun email n'arrive. Par défaut : désactivé. |

## Résultats et métriques

- **Contenu de l'email** : l'email reçu est attaché au résultat du step, vous pouvez le consulter lors de la revue de l'exécution.
- **Timing** : début, fin et durée (en secondes). La durée indique le temps de livraison.
- **Statut** : échec si aucun email n'arrive dans la boîte.

Consultez [Email et SMS dans les tests](/fr/tests/email-sms) pour comprendre le fonctionnement des boîtes de test et leur adressage.
