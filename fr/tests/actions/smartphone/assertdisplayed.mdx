Vérifiez qu'un élément est visible dans l'application, en échouant s'il n'apparaît pas dans le délai imparti.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Élément cible | Oui, sauf si un selector manuel est défini | Description en langage naturel de l'élément qui doit être visible. |
| Selector manuel | Non | Selector XPath ou CSS, utilisé lorsque l'IA est désactivée ou comme selector enregistré. |
| Timeout | Non | Combien de temps attendre l'élément avant l'échec. Par défaut : `30s`. |
| Utilisation de l'IA | Non | `inherit` (par défaut, suit la stratégie du test), `smart` ou `always`. |
| Ignorer l'erreur sur ce step | Non | L'exécution continue avec le step suivant si celui-ci échoue. Par défaut : désactivé. |

## Résultats et métriques

- **Timing** : début, fin et durée (en secondes) enregistrés dans le résultat du step. En cas d'échec, la durée reflète la totalité du timeout.
- **Statut** : succès si l'élément est devenu visible dans le délai imparti, échec sinon.
- **Capture d'écran** : prise pour l'état vérifié.
