Testez un flux de vérification par email de bout en bout. Le step surveille une boîte de test, attend l'email, extrait le code avec l'IA, et le stocke dans une variable que vous pouvez saisir dans le champ de confirmation de l'application.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Email de réception du code | Oui | L'adresse de la boîte de test qui doit recevoir l'email contenant le code. |
| Variable | Oui | Nom de la variable qui reçoit le code. Par défaut : `MailCode`. |
| Ignorer l'erreur sur ce step | Non | L'exécution continue avec le step suivant si aucun code n'est trouvé. Par défaut : désactivé. |

## Résultats et métriques

- **Variable** : le code extrait est stocké dans la variable, prêt à être saisi dans un champ de confirmation par un step Fill input ultérieur.
- **Timing** : début, fin et durée (en secondes). La durée inclut l'attente de réception de l'email.
- **Statut** : échec si aucun email n'arrive ou si l'IA ne trouve pas de code dedans.
- **Utilisation de l'IA** : le coût IA de l'extraction est ajouté à l'exécution.

Consultez [Email et SMS dans les tests](/fr/tests/email-sms) pour comprendre le fonctionnement des boîtes de test.
