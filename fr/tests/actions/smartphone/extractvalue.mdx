Capturez des données de l'application pour une utilisation ultérieure : un numéro de commande à vérifier, un prix à comparer, un identifiant généré à saisir dans un autre formulaire. Le step lit le texte de l'élément (ou sa valeur, pour les champs de formulaire) et le stocke dans une variable.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Élément cible | Oui, sauf si un selector manuel est défini | Description en langage naturel de l'élément à lire. |
| Selector manuel | Non | Selector XPath ou CSS, utilisé lorsque l'IA est désactivée ou comme selector enregistré. |
| Variable | Oui | Nom de la variable qui reçoit la valeur extraite. Par défaut : `Extracted`. |
| Utilisation de l'IA | Non | `inherit` (par défaut, suit la stratégie du test), `smart` ou `always`. |
| Ignorer l'erreur sur ce step | Non | L'exécution continue avec le step suivant si celui-ci échoue. Par défaut : désactivé. |

## Résultats et métriques

- **Variable** : le contenu textuel de l'élément (ou sa valeur, pour les champs de formulaire) est stocké dans la variable et disponible pour tous les steps suivants. Insérez-la partout où un champ supporte les variables.
- **Timing** : début, fin et durée (en secondes) enregistrés dans le résultat du step.
- **Statut** : succès, avertissement ou échec, avec le message d'erreur en cas d'échec.
- **Capture d'écran** : prise après l'exécution du step.
- **Utilisation de l'IA** : si l'IA a localisé l'élément, le selector corrigé est enregistré sur le test et le coût IA est ajouté à l'exécution.

Consultez [Variables](/fr/tests/variables) pour savoir comment les valeurs extraites peuvent être réutilisées entre les steps.
