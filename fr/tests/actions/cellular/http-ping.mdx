---
title: HTTP Ping
description: Mesurez la latence de réponse d'un point d'accès web via la connexion cellulaire.
---

Interrogez une URL de façon répétée et mesurez la latence de réponse. Le plus proche de ce qu'un utilisateur final ressent en accédant à votre service via le réseau mobile.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Hôte | Oui | URL à pinger, par exemple `http://kapptivate.com`. |
| Nombre de pings | Non | Nombre de requêtes. Par défaut : 10. |
| Ignorer le certificat SSL | Non | Ignorer la vérification du certificat. Par défaut : désactivé. |
| Le HTTP Ping doit échouer | Non | Inverse le verdict. |

Ainsi que les [paramètres de connexion](/fr/tests/actions/cellular/overview#paramètres-de-connexion) partagés et la surcharge DNS (`command.override_dns`, `command.dns`).

## Résultats et métriques

Dans le rapport de l'action (`result`) :

| Champ | Description |
|-------|-------------|
| `rtt_mean` | Temps de réponse moyen. |
| `packet_loss_pct` | Pourcentage de requêtes échouées. |
| `ping_count` | Nombre de requêtes. |
| `dns_server` | Serveurs DNS utilisés. |
| `duration` | Temps d'exécution total en secondes. |

Dans Analytics, chaque exécution alimente la mesure HTTP ping avec `rtt_seconds`, `packet_loss_percent` et `duration_seconds`, étiquetée par statut, APN, serveur DNS, antenne et technologie d'accès.
