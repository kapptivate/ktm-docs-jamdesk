title: Wait USSD Push
description: Attendez un message USSD non sollicité envoyé par le réseau.
---

Attendez un USSD push initié par le réseau : une alerte de solde, une notification d'abonnement, un message promotionnel. L'action n'envoie rien ; elle surveille la SIM et vérifie le message à son arrivée.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Message attendu | Non | Le contenu du push que vous attendez. Le résultat enregistre le ratio de correspondance. |
| Vérifier la réception depuis une étape précédente | Non | Commencer la surveillance depuis la position d'une action précédente, pour les push déclenchés plus tôt dans le flux. |
| La réception est optionnelle | Non | Le test réussit même si aucun push n'arrive ; les variables reçoivent leurs valeurs par défaut. Par défaut : désactivé. |

## Résultats et métriques

Dans le rapport de l'action (`result`) :

| Champ | Description |
|-------|-------------|
| `message` | Le contenu du push reçu. |
| `session` | État de la session après le push : `alive` ou `close`. |
| `ratio` | Ratio de correspondance (0 à 100) par rapport au message attendu. |
| `relative_duration` | Temps écoulé depuis le début de la surveillance jusqu'à l'arrivée du push, en secondes. |
| `dynamic_variables` | Valeurs extraites du push dans les variables. |

Dans Analytics, chaque exécution alimente la mesure USSD push avec `reception_duration_seconds` et `duration_seconds`, étiquetée par statut, antenne et technologie d'accès.
