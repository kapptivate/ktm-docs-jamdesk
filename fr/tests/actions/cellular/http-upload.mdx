---
title: HTTP Upload
description: Envoyez un fichier généré via HTTP et mesurez le débit montant.
---

Envoyez un fichier d'une taille choisie vers un point d'accès HTTP et mesurez le débit montant, la direction qui souffre en premier quand une cellule est congestionnée.

## Paramètres

| Paramètre | Champ JSON | Requis | Description |
|-----------|------------|--------|-------------|
| Lien d'envoi | `command.upload_link` | Oui | Le point d'accès qui accepte l'envoi. |
| Taille du fichier | `command.file_size_in_byte` | Oui | Taille du contenu généré, en octets. |
| L'envoi doit échouer | `expect_failure` | Non | Inverse le verdict. |

Ainsi que les [paramètres de connexion](/fr/tests/actions/cellular/overview#paramètres-de-connexion) partagés et la surcharge DNS (`command.override_dns`, `command.dns`).

## Dans le JSON du scénario

```json
{
  "type": "DATA_HTTP_UPLOAD",
  "owner": "SIM_1",
  "skip": false,
  "expect_failure": false,
  "command": {
    "upload_link": "https://files.example.com/upload",
    "file_size_in_byte": 5000000,
    "access_technology": "",
    "default_apn": true,
    "keep_alive": false,
    "check_network": { "active": true },
    "override_dns": false,
    "dns": [],
    "options": { "artifacts": {} }
  },
  "assertions": [],
  "store": []
}
```

## Résultats et métriques

Dans le rapport de l'action (`result`) :

| Champ | Description |
|-------|-------------|
| `throughput_in_kbps` | Débit d'envoi. |
| `md5sum` | Hash du contenu envoyé. |
| `dns_server` | Serveurs DNS utilisés. |
| `duration` | Temps d'exécution total en secondes. |

Dans Analytics, chaque exécution alimente la mesure HTTP upload avec `throughput_kbps` et `duration_seconds`, étiquetée par statut, URL, APN, antenne et technologie d'accès.
