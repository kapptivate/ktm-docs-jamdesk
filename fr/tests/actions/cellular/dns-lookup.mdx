---
title: DNS Lookup
description: Résolvez un nom de domaine via la connexion cellulaire et mesurez le temps de résolution.
---

Résolvez un domaine via la connexion data de la SIM. Vérifiez que le DNS de l'opérateur répond correctement et suivez le temps de résolution, un coupable caché fréquent des expériences mobiles lentes.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Hôte | Oui | Domaine à résoudre, par exemple `kapptivate.com`. |
| Surcharge DNS | Non | Interroger des serveurs DNS spécifiques au lieu de ceux de l'opérateur. |
| Le DNS Lookup doit échouer | Non | Inverse le verdict, pour les domaines qui doivent être bloqués. |

Ainsi que les [paramètres de connexion](/fr/tests/actions/cellular/overview#paramètres-de-connexion) partagés.

## Résultats et métriques

Dans le rapport de l'action (`result`) :

| Champ | Description |
|-------|-------------|
| `ip_addresses` | Les adresses IP résolues. |
| `dns_server` | Les serveurs DNS qui ont répondu. |
| `duration` | Temps de résolution en secondes. |

Dans Analytics, chaque exécution alimente la mesure DNS avec `duration_seconds`, étiquetée par statut, hôte, serveur DNS, APN, antenne et technologie d'accès.
