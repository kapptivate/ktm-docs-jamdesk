---
title: Send Mail
description: Envoyez un e-mail via SMTP sur la connexion cellulaire.
---

Envoyez un e-mail via un serveur SMTP sur la connexion data de la SIM. Vérifiez que l'envoi d'e-mails fonctionne depuis le réseau mobile, y compris à travers le NAT opérateur et le filtrage SMTP.

## Paramètres

| Paramètre | Champ JSON | Requis | Description |
|-----------|------------|--------|-------------|
| Serveur SMTP | `command.smtp_url` | Oui | L'adresse du serveur SMTP. |
| Compte SMTP | `command.smtp_user_email`, `command.smtp_password` | Oui | Identifiants utilisés pour l'envoi. |
| Destinataire | `command.mail_receiver` | Oui | Adresse e-mail de destination. |
| Objet | `command.email_subject` | Non | Objet de l'e-mail. |
| Corps | `command.email_body` | Non | Contenu de l'e-mail. |
| L'envoi doit échouer | `expect_failure` | Non | Inverse le verdict, pour le SMTP qui doit être bloqué. |

Ainsi que les [paramètres de connexion](/fr/tests/actions/cellular/overview#paramètres-de-connexion) partagés.

## Dans le JSON du scénario

```json
{
  "type": "DATA_EMAIL_SEND",
  "owner": "SIM_1",
  "skip": false,
  "expect_failure": false,
  "command": {
    "smtp_url": "smtp.example.com:587",
    "smtp_user_email": "qa@example.com",
    "smtp_password": "secret",
    "mail_receiver": "inbox@example.com",
    "email_subject": "Cellular SMTP check",
    "email_body": "Sent from a real SIM",
    "access_technology": "",
    "default_apn": true,
    "keep_alive": false,
    "check_network": { "active": true },
    "options": { "artifacts": {} }
  },
  "assertions": [],
  "store": []
}
```

## Résultats et métriques

Dans le rapport de l'action (`result`) : le `status` d'envoi et la `duration` en secondes.

Dans Analytics, chaque exécution alimente la mesure d'envoi d'e-mail avec `duration_seconds`, étiquetée par statut, APN, antenne et technologie d'accès.
