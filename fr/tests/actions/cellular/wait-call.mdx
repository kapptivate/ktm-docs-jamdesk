title: Wait Call
description: Attendez un appel entrant sur une vraie SIM, décrochez et évaluez la qualité audio.

Attendez un appel entrant sur la SIM et décrochez. Associé à [Place Call](/fr/tests/actions/cellular/place-call) sur une autre SIM, il vérifie le chemin vocal complet ; avec l'enregistrement et le MOS activés, il évalue la qualité audio réellement entendue par le destinataire.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Technologie d'accès | Non | Technologie réseau, y compris les profils VoLTE. Par défaut : automatique. |
| Temps d'attente | Non | Temps d'attente pour l'appel. Par défaut : 30. |
| Enregistrer l'appel | Non | Enregistrer l'audio reçu. Par défaut : désactivé. |
| Calculer le MOS audio | Non | Évaluer l'audio enregistré par rapport au fichier de référence joué par l'appelant. Nécessite l'enregistrement. |
| Bloquer jusqu'à réception de l'appel | Non | Exécuter de manière synchrone (bloquer le scénario) ou surveiller en arrière-plan. |
| La réception de l'appel est optionnelle | Non | Le test réussit même si aucun appel n'arrive. Par défaut : désactivé. |

## Résultats et métriques

Dans le rapport de l'action (`result`) :

| Champ | Description |
|-------|-------------|
| `seconds_elapsed_until_accept` | Temps avant l'arrivée et le décrochage de l'appel. |
| `call_duration_seconds` | Durée de l'appel maintenu. |
| `caller_number` | Le numéro de l'appelant. |
| `audio_mos` | Le score de qualité audio (MOS), quand l'enregistrement et le MOS sont activés. |
| `ceer_response` | Le rapport de fin d'appel du réseau. |
| `access_technology_report` | La technologie utilisée pour l'appel. |

Dans Analytics, chaque exécution alimente la mesure d'appel avec `cellular_call_duration_seconds`, `cellular_audio_mos` et `duration_seconds`, étiquetée par statut, numéro de l'appelant, antenne et technologie d'accès. `cellular_audio_mos` est la série à surveiller pour la dégradation de la qualité vocale.
