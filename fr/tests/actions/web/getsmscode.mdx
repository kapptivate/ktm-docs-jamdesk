Testez un flux de vérification par SMS de bout en bout. Le step surveille un numéro de téléphone de test, attend l'arrivée du SMS, extrait le code avec l'IA et le stocke dans une variable que vous pouvez saisir dans le champ de confirmation.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Numéro de téléphone destinataire | Oui | Le numéro de téléphone de test qui doit recevoir le SMS. |
| Variable | Oui | Nom de la variable qui reçoit le code. Par défaut : `SmsCode`. |
| Ignorer l'erreur sur ce step | Non | L'exécution continue avec le step suivant si aucun code n'est trouvé. Par défaut : désactivé. |

## Résultats et métriques

- **Variable** : le code extrait est stocké dans la variable, prêt pour un step [Fill input](/fr/tests/actions/web/input).
- **Temps d'exécution** : début, fin et durée (en secondes). La durée inclut l'attente de l'arrivée du SMS.
- **Statut** : échec si aucun SMS n'arrive ou si l'IA ne trouve pas de code dedans.
- **Utilisation de l'IA** : le coût IA de l'extraction est ajouté à l'exécution.

Consultez [Email & SMS dans les tests](/fr/tests/email-sms) pour savoir comment les numéros de téléphone de test sont provisionnés pour votre espace de travail.
