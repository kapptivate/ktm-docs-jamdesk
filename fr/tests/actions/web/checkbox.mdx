---
title: Cocher une case
description: Step obsolète qui coche ou décoche une case à cocher. Utilisez Click à la place.
---

<Danger>
  **Obsolète.** Ce step n'apparaît plus dans la palette de steps. Les tests existants qui l'utilisent continuent de fonctionner. Pour les nouveaux tests, utilisez [Click](/fr/tests/actions/web/click) sur la case à cocher. Un clic bascule l'état au lieu de le définir, vérifiez donc le résultat avec [Vérifier un élément](/fr/tests/actions/web/verifyelement) si nécessaire.
</Danger>

Définissez l'état d'une case à cocher. Le step ne clique que lorsque l'état actuel diffère de celui demandé, il peut donc être répété en toute sécurité.

## Paramètres

| Paramètre | Champ JSON | Requis | Description |
|-----------|------------|--------|-------------|
| Élément cible | `locator.description` | Oui, sauf si un selector manuel est défini | Description en langage naturel de la case à cocher. |
| Selector manuel | `locator.selector` | Non | Selector XPath ou CSS, utilisé lorsque l'IA est désactivée ou comme selector enregistré. |
| Définir comme cochée | `checked` | Non | L'état cible. Par défaut : désactivé (non cochée). |
| Utilisation IA | `ai_strategy` | Non | `inherit` (par défaut, suit la stratégie du test), `smart`, ou `always`. |
| Ignorer l'erreur sur ce step | `ignore_errors` | Non | L'exécution continue avec le step suivant si celui-ci échoue. Par défaut : désactivé. |

## Dans le JSON du scénario

Le step se trouve dans `step_groups[].steps` du scénario de test.

```json
{
  "id": "step-1",
  "type": "checkbox",
  "version": "v1.0.0",
  "disabled": false,
  "inputs": {
    "locator": {
      "selector": [{ "type": "value", "data": { "value": "" } }],
      "description": [{ "type": "value", "data": { "value": "The terms and conditions checkbox" } }]
    },
    "checked": true,
    "enable_ai": true,
    "ai_strategy": "inherit",
    "ignore_errors": false
  },
  "use_capabilities": false,
  "capabilities": {}
}
```

## Résultats et métriques

- **Temps** : début, fin et durée (en secondes) enregistrés dans le résultat du step.
- **Statut** : succès, avertissement ou échec, avec le message d'erreur en cas d'échec.
- **Capture d'écran** : prise après l'exécution du step.
