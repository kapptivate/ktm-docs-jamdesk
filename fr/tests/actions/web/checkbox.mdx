title: Cocher une case
description: Step obsolète qui coche ou décoche une case à cocher. Utilisez Click à la place.
---

<Danger>
  **Obsolète.** Ce step n'apparaît plus dans la palette de steps. Les tests existants qui l'utilisent continuent de fonctionner. Pour les nouveaux tests, utilisez [Click](/fr/tests/actions/web/click) sur la case à cocher. Un clic bascule l'état au lieu de le définir, vérifiez donc le résultat avec [Vérifier un élément](/fr/tests/actions/web/verifyelement) si nécessaire.
</Danger>

Définissez l'état d'une case à cocher. Le step ne clique que lorsque l'état actuel diffère de celui demandé, il peut donc être répété en toute sécurité.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Élément cible | Oui, sauf si un selector manuel est défini | Description en langage naturel de la case à cocher. |
| Selector manuel | Non | Selector XPath ou CSS, utilisé lorsque l'IA est désactivée ou comme selector enregistré. |
| Définir comme cochée | Non | L'état cible. Par défaut : désactivé (non cochée). |
| Utilisation IA | Non | `inherit` (par défaut, suit la stratégie du test), `smart`, ou `always`. |
| Ignorer l'erreur sur ce step | Non | L'exécution continue avec le step suivant si celui-ci échoue. Par défaut : désactivé. |

## Résultats et métriques

- **Temps** : début, fin et durée (en secondes) enregistrés dans le résultat du step.
- **Statut** : succès, avertissement ou échec, avec le message d'erreur en cas d'échec.
- **Capture d'écran** : prise après l'exécution du step.
