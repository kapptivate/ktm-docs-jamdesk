Téléversez un fichier via un élément `<input type="file">`. Vous dirigez le step vers le champ de téléversement et lui fournissez l'URL du fichier à envoyer ; la plateforme télécharge le fichier et le transmet au champ, comme si un utilisateur l'avait sélectionné depuis son disque.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Élément cible | Oui, sauf si un selector manuel est défini | Le champ de téléversement (ou son contrôle visible). L'IA utilise la description pour le trouver. |
| URL du fichier | Oui | URL du fichier à téléverser. Le fichier doit être accessible depuis l'agent qui exécute le test. Supporte les variables. |
| Selector manuel | Non | Selector XPath ou CSS, utilisé quand l'IA est désactivée ou comme selector enregistré. |
| Utilisation de l'IA | Non | `inherit` (par défaut, suit la stratégie du test), `smart` ou `always`. |
| Ignorer l'erreur sur ce step | Non | L'exécution continue avec le step suivant si celui-ci échoue. Par défaut : désactivé. |

## Résultats et métriques

- **Temps d'exécution** : début, fin et durée (en secondes), incluant le temps de téléchargement et d'envoi du fichier.
- **Statut** : succès, avertissement ou échec, avec le message d'erreur en cas d'échec.
- **Capture d'écran** : prise après le téléversement.
- **Utilisation de l'IA** : si l'IA a localisé l'élément, le selector corrigé est enregistré sur le test et le coût IA est ajouté à l'exécution.
