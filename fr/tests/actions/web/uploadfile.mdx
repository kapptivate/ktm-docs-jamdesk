title: Upload file

Téléversez un fichier via un élément `<input type="file">`. Vous dirigez le step vers le champ de téléversement et lui fournissez l'URL du fichier à envoyer ; la plateforme télécharge le fichier et le transmet au champ, comme si un utilisateur l'avait sélectionné depuis son disque.

## Paramètres

| Paramètre | Champ JSON | Requis | Description |
|-----------|------------|--------|-------------|
| Élément cible | `locator.description` | Oui, sauf si un selector manuel est défini | Le champ de téléversement (ou son contrôle visible). L'IA utilise la description pour le trouver. |
| URL du fichier | `url` | Oui | URL du fichier à téléverser. Le fichier doit être accessible depuis l'agent qui exécute le test. Supporte les variables. |
| Selector manuel | `locator.selector` | Non | Selector XPath ou CSS, utilisé quand l'IA est désactivée ou comme selector enregistré. |
| Utilisation de l'IA | `ai_strategy` | Non | `inherit` (par défaut, suit la stratégie du test), `smart` ou `always`. |
| Ignorer l'erreur sur ce step | `ignore_errors` | Non | L'exécution continue avec le step suivant si celui-ci échoue. Par défaut : désactivé. |

## Dans le scénario JSON

Le step se trouve dans `step_groups[].steps` du scénario de test.

```json
{
  "id": "step-1",
  "type": "upload-file",
  "version": "v1.0.0",
  "disabled": false,
  "inputs": {
    "locator": {
      "selector": [{ "type": "value", "data": { "value": "" } }],
      "description": [{ "type": "value", "data": { "value": "The attachment file input in the claim form" } }]
    },
    "url": [{ "type": "value", "data": { "value": "https://files.example.com/fixtures/id-card.png" } }],
    "enable_ai": true,
    "ai_strategy": "inherit",
    "ignore_errors": false
  },
  "use_capabilities": false,
  "capabilities": {}
}
```

## Résultats et métriques

- **Temps d'exécution** : début, fin et durée (en secondes), incluant le temps de téléchargement et d'envoi du fichier.
- **Statut** : succès, avertissement ou échec, avec le message d'erreur en cas d'échec.
- **Capture d'écran** : prise après le téléversement.
- **Utilisation de l'IA** : si l'IA a localisé l'élément, le selector corrigé est enregistré sur le test et le coût IA est ajouté à l'exécution.
