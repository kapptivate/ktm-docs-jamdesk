Suivez un lien d'e-mail dans le cadre du flux : confirmation de compte, réinitialisation de mot de passe, connexion par lien magique. Le step attend l'e-mail, demande à l'IA le lien correspondant à votre intention et le stocke dans une variable. Ouvrez-le avec un step [Navigate to URL](/fr/tests/actions/web/navigatetourl) pour continuer le scénario.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| E-mail destinataire | Oui | L'adresse de la boîte mail de test qui doit recevoir l'e-mail contenant le lien. |
| Variable | Oui | Nom de la variable qui reçoit le lien. Par défaut : `MailLink`. |
| Intention | Non | Quel lien l'IA doit extraire. Par défaut : `The first link in the text`. |
| Ignorer l'erreur sur ce step | Non | L'exécution continue avec le step suivant si aucun lien n'est trouvé. Par défaut : désactivé. |

Utilisez l'intention pour désambiguïser les e-mails contenant plusieurs liens : "The password reset link", "The link in the Confirm my account button".

## Résultats et métriques

- **Variable** : le lien extrait est stocké dans la variable. Ouvrez-le avec un step de navigation pour continuer le flux.
- **Temps d'exécution** : début, fin et durée (en secondes). La durée inclut l'attente de l'arrivée de l'e-mail.
- **Statut** : échec si aucun e-mail n'arrive ou si l'IA ne trouve pas de lien correspondant à l'intention.
- **Utilisation de l'IA** : le coût IA de l'extraction est ajouté à l'exécution.

Consultez [Email & SMS dans les tests](/fr/tests/email-sms) pour savoir comment les boîtes mail de test fonctionnent.
