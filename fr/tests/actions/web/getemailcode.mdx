Testez un flux de vérification par e-mail de bout en bout. Le step surveille une boîte mail de test, attend l'e-mail, extrait le code avec l'IA et le stocke dans une variable que vous pouvez saisir dans le champ de confirmation.

## Paramètres

| Paramètre | Champ JSON | Requis | Description |
|-----------|------------|--------|-------------|
| E-mail destinataire | `mailbox` | Oui | L'adresse de la boîte mail de test qui doit recevoir l'e-mail contenant le code. |
| Variable | `variable` | Oui | Nom de la variable qui reçoit le code. Par défaut : `MailCode`. |
| Ignorer l'erreur sur ce step | `ignore_errors` | Non | L'exécution continue avec le step suivant si aucun code n'est trouvé. Par défaut : désactivé. |

## Dans le scénario JSON

Le step se trouve dans `step_groups[].steps` du scénario de test.

```json
{
  "id": "step-1",
  "type": "ai-get-mail-code",
  "version": "v1.0.0",
  "disabled": false,
  "inputs": {
    "mailbox": [{ "type": "value", "data": { "value": "qa-signup@example.com" } }],
    "variable": "MailCode",
    "ignore_errors": false
  },
  "use_capabilities": false,
  "capabilities": {}
}
```

## Résultats et métriques

- **Variable** : le code extrait est stocké dans la variable, prêt à être saisi dans un champ de confirmation par un step Fill input ultérieur.
- **Temps d'exécution** : début, fin et durée (en secondes). La durée inclut l'attente de l'arrivée de l'e-mail.
- **Statut** : échec si aucun e-mail n'arrive ou si l'IA ne trouve pas de code dedans.
- **Utilisation de l'IA** : le coût IA de l'extraction est ajouté à l'exécution.

Consultez [Email & SMS dans les tests](/fr/tests/email-sms) pour savoir comment les boîtes mail de test fonctionnent.
