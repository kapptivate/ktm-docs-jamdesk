Faites défiler la page jusqu'à ce qu'un élément soit visible. Utilisez ce step avant d'interagir avec du contenu sous la ligne de flottaison, ou pour vérifier que les sections en chargement différé se chargent effectivement lorsqu'on les atteint.

## Paramètres

| Paramètre | Champ JSON | Requis | Description |
|-----------|------------|--------|-------------|
| Élément cible | `locator.description` | Oui, sauf si un selector manuel est défini | Description en langage naturel de l'élément vers lequel défiler. |
| Selector manuel | `locator.selector` | Non | Selector XPath ou CSS, utilisé quand l'IA est désactivée ou comme selector enregistré. |
| Comportement du défilement | `behavior` | Non | `auto` (par défaut), `smooth` ou `instant`. |
| Positionnement vertical | `block` | Non | Position verticale de l'élément : `start`, `center` (par défaut), `end` ou `nearest`. |
| Positionnement horizontal | `inline` | Non | Position horizontale de l'élément : `start`, `center` (par défaut), `end` ou `nearest`. |
| Utilisation de l'IA | `ai_strategy` | Non | `inherit` (par défaut, suit la stratégie du test), `smart` ou `always`. |
| Ignorer l'erreur sur ce step | `ignore_errors` | Non | L'exécution continue avec le step suivant si celui-ci échoue. Par défaut : désactivé. |

## Dans le scénario JSON

Le step se trouve dans `step_groups[].steps` du scénario de test.

```json
{
  "id": "step-1",
  "type": "scroll",
  "version": "v1.0.0",
  "disabled": false,
  "inputs": {
    "locator": {
      "selector": [{ "type": "value", "data": { "value": "" } }],
      "description": [{ "type": "value", "data": { "value": "The pricing section at the bottom of the page" } }]
    },
    "behavior": "auto",
    "block": "center",
    "inline": "center",
    "enable_ai": true,
    "ai_strategy": "inherit",
    "ignore_errors": false
  },
  "use_capabilities": false,
  "capabilities": {}
}
```

## Résultats et métriques

- **Temps d'exécution** : début, fin et durée (en secondes) enregistrés dans le résultat du step.
- **Statut** : succès, avertissement ou échec, avec le message d'erreur en cas d'échec.
- **Capture d'écran** : prise après l'exécution du step, montrant la nouvelle position.
- **Utilisation de l'IA** : si l'IA a localisé l'élément, le selector corrigé est enregistré sur le test et le coût IA est ajouté à l'exécution.
