Les tests web sont construits à partir des steps ci-dessous. Les groupes correspondent aux catégories de la palette **Ajouter un step** dans le [test builder](/fr/tests/builder), ce que vous voyez ici est donc ce que vous voyez dans le produit.

| Step | Ce qu'il fait |

| [Click](/fr/tests/actions/web/click) | Cliquer sur un élément localisé par description IA ou selector |
| [Double click](/fr/tests/actions/web/doubleclick) | Deux clics rapides, pour les modes édition et les éléments de liste |
| [Right click](/fr/tests/actions/web/rightclick) | Ouvrir le menu contextuel d'un élément |
| [Hover](/fr/tests/actions/web/hover) | Révéler des menus, infobulles et états de survol |
| [Upload file](/fr/tests/actions/web/uploadfile) | Fournir un fichier à un champ de téléversement |
| [Press keys](/fr/tests/actions/web/keys) | Envoyer des touches et raccourcis clavier |
| [Fill input](/fr/tests/actions/web/input) | Saisir du texte, des variables ou des données générées dans un champ |
| [Scroll](/fr/tests/actions/web/scroll) | Faire défiler jusqu'à rendre un élément visible |

## Paramètres communs des steps

Chaque step partage quelques paramètres, affichés dans le panneau latéral du step dans le builder :

- **Ignorer l'erreur sur ce step** (`ignore_errors`) : l'exécution continue avec le step suivant même si celui-ci échoue.
- **Ignorer pendant l'exécution** (`disabled`) : le step reste dans le scénario mais n'est pas exécuté.
- **Capacités personnalisées** (`use_capabilities`, `capabilities`) : ajuster ou remplacer le comportement d'exécution pour ce step uniquement.

Les steps qui ciblent un élément partagent également les paramètres IA : une **description** en langage naturel, un **selector manuel** optionnel, et une **stratégie IA** (`inherit`, `smart` ou `always`) qui détermine quand l'IA localise l'élément par rapport au selector enregistré. Consultez [Sélecteurs IA](/fr/ai/selectors).

## Vérifications de step

Les steps d'interaction et de navigation peuvent porter des **vérifications**, évaluées juste après l'exécution du step :

- **URL** : l'adresse de la page correspond (ou ne correspond pas) à un motif.
- **Temps de chargement** : le chargement de la page reste sous un seuil.
- **Durée du step** : le step lui-même reste sous un seuil.

Chaque vérification est définie en mode **erreur** (un échec de la vérification fait échouer le step) ou **avertissement** (le step réussit mais est signalé). Les valeurs attendues et réelles apparaissent dans les résultats de l'exécution.

## Des résultats à l'analytique

Chaque step enregistre son temps d'exécution, son statut, ses captures d'écran, les résultats de ses vérifications et toutes les variables qu'il a créées. Au niveau du test, chaque exécution alimente la mesure `test` utilisée par [Analytics](/fr/analytics/dashboards) : durée, compteurs de succès/avertissement/échec, et en cas d'échec, quel type de step a causé la défaillance. Les métriques personnalisées créées dans les steps remontent au même endroit.
