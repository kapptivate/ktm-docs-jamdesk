Saisissez du texte dans un champ de la page. La valeur peut mélanger du texte fixe, des variables et des données générées, pour qu'un même step puisse se connecter avec un compte enregistré ou s'inscrire avec un e-mail aléatoire à chaque exécution.

## Paramètres

| Paramètre | Champ JSON | Requis | Description |
|-----------|------------|--------|-------------|
| Élément cible | `locator.description` | Oui, sauf si un selector manuel est défini | Description en langage naturel du champ. L'IA l'utilise pour trouver l'élément. |
| Selector manuel | `locator.selector` | Non | Selector XPath ou CSS, utilisé quand l'IA est désactivée ou comme selector enregistré. |
| Valeur | `value` | Oui | Texte à saisir. Supporte les variables et les fonctions comme le texte aléatoire. |
| Utilisation de l'IA | `ai_strategy` | Non | `inherit` (par défaut, suit la stratégie du test), `smart` ou `always`. |
| Activer l'IA | `enable_ai` | Non | Quand désactivé, seul le selector manuel est utilisé. Par défaut : activé. |
| Ignorer l'erreur sur ce step | `ignore_errors` | Non | L'exécution continue avec le step suivant si celui-ci échoue. Par défaut : désactivé. |

## Dans le scénario JSON

Le step se trouve dans `step_groups[].steps` du scénario de test.

```json
{
  "id": "step-1",
  "type": "input",
  "version": "v1.0.0",
  "disabled": false,
  "inputs": {
    "locator": {
      "selector": [{ "type": "value", "data": { "value": "" } }],
      "description": [{ "type": "value", "data": { "value": "The email field of the login form" } }]
    },
    "value": [
      { "type": "value", "data": { "value": "qa+" } },
      { "type": "test_variable", "data": { "name": "UserId" } },
      { "type": "value", "data": { "value": "@example.com" } }
    ],
    "enable_ai": true,
    "ai_strategy": "inherit",
    "ignore_errors": false
  },
  "use_capabilities": false,
  "capabilities": {}
}
```

L'exemple de `value` ci-dessus mélange du texte brut avec une variable de test. Consultez [Variables](/fr/tests/variables) pour tout ce que vous pouvez insérer : variables de test, variables globales et fonctions comme le texte aléatoire.

## Résultats et métriques

- **Temps d'exécution** : début, fin et durée (en secondes) enregistrés dans le résultat du step.
- **Statut** : succès, avertissement ou échec, avec le message d'erreur en cas d'échec.
- **Capture d'écran** : prise après l'exécution du step.
- **Utilisation de l'IA** : si l'IA a localisé l'élément, le selector corrigé est enregistré sur le test et le coût IA est ajouté à l'exécution.
