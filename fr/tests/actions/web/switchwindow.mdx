Poursuivez le test dans un autre onglet ou une autre fenêtre : une page de paiement ouverte dans un nouvel onglet, une popup OAuth, un aperçu de document. Chaque step après celui-ci s'exécute dans la fenêtre vers laquelle vous avez basculé.

## Paramètres

| Paramètre | Champ JSON | Requis | Description |
|-----------|------------|--------|-------------|
| Nom de l'onglet ou URL | `matcher` | Oui | Texte comparé au titre et à l'URL des onglets ouverts. Le premier onglet correspondant devient l'onglet actif. Supporte les variables. |
| Ignorer l'erreur sur ce step | `ignore_errors` | Non | L'exécution continue avec le step suivant si aucun onglet ne correspond. Par défaut : désactivé. |

## Dans le JSON du scénario

Le step se trouve dans `step_groups[].steps` du scénario de test.

```json
{
  "id": "step-1",
  "type": "switch-window",
  "version": "v1.0.0",
  "disabled": false,
  "inputs": {
    "matcher": [{ "type": "value", "data": { "value": "checkout.stripe.com" } }],
    "ignore_errors": false
  },
  "use_capabilities": false,
  "capabilities": {}
}
```

## Résultats et métriques

- **Temps** : début, fin et durée (en secondes) enregistrés dans le résultat du step.
- **Statut** : échec si aucun onglet ouvert ne correspond au nom ou à l'URL.
- **Capture d'écran** : prise dans l'onglet nouvellement actif.
