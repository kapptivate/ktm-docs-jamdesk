Cliquez sur un élément de la page : un bouton, un lien, une entrée de menu. Décrivez l'élément en langage naturel et l'IA le trouve, ou ciblez-le vous-même avec un selector XPath ou CSS.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Élément cible | Oui, sauf si un selector manuel est défini | Description en langage naturel de l'élément. L'IA l'utilise pour trouver l'élément, par exemple "Le bouton Se connecter dans le header". |
| Selector manuel | Non | Selector XPath ou CSS, utilisé quand l'IA est désactivée ou comme selector enregistré. |
| Utilisation de l'IA | Non | `inherit` (par défaut, suit la stratégie du test), `smart` ou `always`. |
| Activer l'IA | Non | Quand désactivé, seul le selector manuel est utilisé. Par défaut : activé. |
| Ignorer l'erreur sur ce step | Non | L'exécution continue avec le step suivant si celui-ci échoue. Par défaut : désactivé. |

Le fonctionnement de la résolution IA de l'élément, et les cas de repli sur le selector enregistré, sont détaillés dans [Sélecteurs IA](/fr/ai/selectors).

## Résultats et métriques

- **Temps d'exécution** : début, fin et durée (en secondes) enregistrés dans le résultat du step.
- **Statut** : succès, avertissement ou échec, avec le message d'erreur en cas d'échec.
- **Capture d'écran** : prise après le clic, pour voir la page vers laquelle il a mené.
- **Vérifications de step** : les vérifications d'URL, de temps de chargement et de durée du step rapportent leurs valeurs attendues et réelles lorsqu'elles sont configurées.
- **Utilisation de l'IA** : si l'IA a localisé l'élément, le selector corrigé est enregistré sur le test et le coût IA est ajouté à l'exécution.
