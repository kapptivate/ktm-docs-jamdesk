<Danger>
  **Obsolète.** Ce step n'apparaît plus dans la palette de steps. Les tests existants qui l'utilisent continuent de fonctionner. Pour les nouveaux tests, utilisez [Vérifier un élément](/fr/tests/actions/web/verifyelement) avec l'état **displayed**.
</Danger>

Vérifiez qu'un élément est visible sur la page, en échouant s'il n'apparaît pas dans le délai imparti.

## Paramètres

| Paramètre | Requis | Description |
| ----------- | -------- | ------------- |
| Élément cible | Oui, sauf si un selector manuel est défini | Description en langage naturel de l'élément qui doit être visible. |
| Selector manuel | Non | Selector XPath ou CSS, utilisé lorsque l'IA est désactivée ou comme selector enregistré. |
| Délai d'attente | Non | Durée d'attente avant l'échec. Par défaut : `30s`. |
| Utilisation IA | Non | `inherit` (par défaut, suit la stratégie du test), `smart`, ou `always`. |
| Ignorer l'erreur sur ce step | Non | L'exécution continue avec le step suivant si celui-ci échoue. Par défaut : désactivé. |

## Résultats et métriques

- **Temps** : début, fin et durée (en secondes) enregistrés dans le résultat du step. En cas d'échec, la durée reflète le délai d'attente complet.
- **Statut** : succès si l'élément est devenu visible dans le délai imparti, échec dans le cas contraire.
- **Capture d'écran** : prise pour l'état vérifié.
