Les tests UI cassent pour de mauvaises raisons : un changement de DOM renomme un selector, un code à usage unique arrive dans une boîte de réception, un bug visuel passe à travers une assertion en correspondance exacte. Kapptivate intègre l'IA dans le cycle de vie des tests pour absorber précisément ce type de fragilité, tout en vous laissant le contrôle sur son périmètre d'action.

Vous décidez de l'empreinte : l'IA peut être complètement désactivée, limitée à des steps individuels, ou piloter chaque selector, par test, avec une [stratégie](/fr/ai/selectors#ai-strategies) qui définit quand elle s'engage.

## Où vous le contrôlez

- **À la création** : la case **AI usage** active l'IA dès le premier step ([Créer un test](/fr/tests/create)).
- **Par test** : le toggle **Enable AI** et la stratégie dans **Test settings > AI** ([Test builder](/fr/tests/builder)).
- **Par step** : le toggle **Turn on AI for this step** surcharge le paramètre du test là où vous en avez besoin.
