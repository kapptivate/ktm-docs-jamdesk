## Paramètres

| Champ | Type | Requis | Description |
| --- | --- | --- | --- |
| `creator` | string | Non | Filtrer par identifiant(s) d'utilisateur créateur, séparés par des virgules |
| `detail` | boolean | Non | Renvoie le JSON complet au lieu de lignes compactes (défaut : false) |
| `ids` | string | Non | Identifiants de test campaigns séparés par des virgules |
| `page` | number | Non | Numéro de page (défaut 1) |
| `page_size` | number | Non | Éléments par page (défaut 50, max 4000) |
| `product` | string | Non | Filtrer par slug de produit ou short_name (liste séparée par des virgules acceptée) |

## Description

```text
A Tests campaign is a SAVED, ordered pack of tests attached to a single product — use it to group regression scenarios, smoke packs, or anything you run as one unit. To run one: build_campaign(test_campaign_id=<id>) → run_campaign(overrides=…).

Returns compact one-liners by default (`"#42 Regression pack [8 tests, valid]"`). detail=true returns the full Tests campaign including the test_list.
```
