Examiner en détail le(s) résultat(s) d'une exécution de monitoring (planifiée).

<Note>
  **Lecture seule · Appelle des systèmes externes.** Appel sûr : cet outil ne modifie pas les données.
</Note>

## Paramètres

| Champ | Type | Requis | Description |
| --- | --- | --- | --- |
| `include_actions` | boolean | Non | Inclure les résultats bruts des actions avec captures d'écran, artefacts, logs (par défaut : false). Augmente significativement la taille de la réponse. Note : les captures d'écran incluent des URL complètes ; les chemins d'artefacts sont relatifs, préfixez https://&lt;KAPPTIVATE_HOST&gt;/robot/ pour construire l'URL complète. |
| `include_full_json` | boolean | Non | Inclure le blob JSON brut complet du résultat (par défaut : false). À utiliser uniquement lorsque vous avez besoin des métadonnées de campagne, des options ou de l'historique. |
| `result_ids` | string | Oui | Identifiants de résultats de monitoring séparés par des virgules (depuis list_monitoring_results). Un seul ID pour un résultat, plusieurs pour une investigation groupée. |

## Exemple d'arguments

Arguments illustratifs qu'un agent fournit lors de l'appel de cet outil :

```json
{
  "result_ids": "string"
}
```

## Description

```text
Renvoie la raison de l'échec, le statut par test/par action, la durée et les variables. Supporte le traitement par lot : passez des identifiants séparés par des virgules pour examiner plusieurs échecs en un seul appel. Utilisez include_actions=true pour les résultats bruts des actions (captures d'écran, artefacts, logs).

Les identifiants de résultat proviennent de list_monitoring_results. Filtrez toujours list_monitoring_results par monitoring_id d'abord pour vous assurer que les résultats appartiennent au bon opérateur. Pour les exécutions de campagne ou de Tests-campagne, utilisez get_execution_detail à la place.
```
