## Paramètres

| Champ | Type | Requis | Description |
| --- | --- | --- | --- |
| `archived` | boolean | Non | Filtrer par statut archivé |
| `detail` | boolean | Non | Renvoie le JSON complet au lieu de résumés compacts (défaut : false) |
| `name` | string | Non | Filtrer par nom de test |
| `operator` | string | Non | Filtrer par slug de l'operator |
| `ordering` | string | Non | Ordre de tri (ex. 'name', '-name', 'id', '-id') |
| `page` | number | Non | Numéro de page (défaut : 1) |
| `page_size` | number | Non | Nombre de tests par page (défaut : 20) |
| `product` | string | Non | Filtrer par slug de produit (format : operator~product) |
| `tags` | string | Non | Filtrer par tags (séparés par des virgules) |

## Exemple d'arguments

Arguments illustratifs qu'un agent fournit lors de l'appel de cet outil :

```json
{
  "archived": false,
  "detail": false
}
```

## Description

```text
Returns compact summaries by default; use detail=true for full JSON. List test scenarios. Tests belong to a product and can be executed via campaigns.

Flat discovery: returns all tests in a product (paginated, filterable by name/tag/status). For folder-based browsing, use list_collections → list_collection_items instead.
```
