Liste les ressources (SIM, web agents, smartphones) pour l'exécution et la création de tests.

<Note>
  **Lecture seule · Appelle des systèmes externes.** Appel sûr : cet outil ne modifie pas les données.
</Note>

## Paramètres

| Champ | Type | Requis | Description |
| --- | --- | --- | --- |
| `available` | boolean | Non | Filtrer par statut en ligne/disponible (défaut : true). Passez false pour inclure les ressources hors ligne. |
| `configured` | boolean | Non | Filtrer par statut configuré (défaut : true). Configuré signifie : SIM a un national_msisdn, agent a un hostname, smartphone a des props. |
| `detail` | boolean | Non | Renvoie le JSON complet au lieu de résumés compacts (défaut : false) |
| `product` | string | Oui | Slug du produit (requis, format : operator~product) |
| `type` | string (sim \| web-agent \| smartphone) | Non | Filtre par type de ressource : sim, web-agent, smartphone (omettez pour les trois) |

## Exemple d'arguments

Arguments illustratifs qu'un agent fournit lors de l'appel de cet outil :

```json
{
  "product": "string"
}
```

## Description

```text
Returns compact summaries by default; use detail=true for full JSON. List resources (SIMs, web agents, smartphones) for test execution and test creation. Resource identifiers become action owners: SIM→national_msisdn, web agent→hostname, smartphone→UUID. Defaults to online/available and configured resources. To list installable apps on a specific smartphone, use get_smartphone_apps instead.
```
