Affiche un aperçu du diff.

<Note>
  **Lecture seule · Appelle des systèmes externes.** Appel sûr : cet outil ne modifie pas les données.
</Note>

## Paramètres

| Champ | Type | Requis | Description |
| --- | --- | --- | --- |
| `actions` | string | Non | Tableau JSON d'actions (remplace les actions actuelles) |
| `auto_correct_kinds` | boolean | Non | Réécrit les kinds d'input_variable déclarés pour correspondre aux types de variables de la plateforme. Défaut false : les incohérences de kind sont des erreurs bloquantes. |
| `description` | string | Non | Nouvelle description du test (uniquement si modification explicite) |
| `name` | string | Non | Nouveau nom du test (uniquement si modification explicite) |
| `test_id` | number | Oui | Identifiant du test pour lequel prévisualiser les modifications |
| `variables` | string | Non | Objet JSON variables (remplace les variables actuelles) |
| `variants` | array | Non | Spécificateurs de variant pin pour la résolution de bucket V3. Chaque élément est '5' (ID numérique) ou 'API:Staging' (group:variant par nom). Mode édition : les groupes non épinglés conservent leurs buckets existants. |

## Description

```text
Protected fields (cannot change): product_id, collection_uid.

Preview the impact of editing a test WITHOUT applying changes. Returns which tests would be invalidated and which monitorings would be deactivated.

V3 buckets: pass variants=["API:Staging", ...] (repeatable, same forms as create_test) to preview re-pinning. In edit mode, groups not pinned retain their existing buckets. Pre-V3 tests get default variants filled in automatically.

Workflow: get_full_test → preview_test_edit → (review impact) → edit_test.
```
