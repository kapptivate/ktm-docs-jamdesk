Exécute une campaign et attend les résultats.

<Warning>
  **Destructif · Écrit des données · Appelle des systèmes externes.** Cet outil peut modifier ou supprimer des données. Vérifiez les arguments avant de laisser un agent l'appeler.
</Warning>

## Paramètres

| Champ | Type | Requis | Description |
| --- | --- | --- | --- |
| `campaign_json` | string | Oui | Template JSON de la campaign (depuis build_campaign) |
| `ignore_errors` | boolean | Non | Ignore les échecs pour tous les types d'action (options.ignore_failed = &lt;all action codes&gt;). Les statuts par test peuvent être réécrits en success. |
| `max_poll_duration` | string | Non | Durée maximale d'attente des résultats (défaut : 10m). Format : chaîne de durée Go (ex. 5m, 15m) |
| `overrides` | string | Non | Objet JSON mappant les identifiants de test à leurs valeurs de variables et assignations d'owners. Format : \{"&lt;test_id&gt;": \{"input_variables": \{...\}, "direct_owners": \{...\}\}\}. direct_owners mappe l'identifiant de device à la valeur (ex. \{"sim_owner": "SIM-001"\}). V3 : les valeurs d'input_variables peuvent être soit des chaînes plates (legacy) soit des objets \{value, bucket\} pour cibler un variant spécifique (variables_v3.md Bucket System). |
| `parallel` | boolean | Non | Exécute les tests de la campaign en parallèle (options.parallel_test_execution=-1). Superadmin uniquement ; le backend rejette les listes de tests qui utilisent des actions SIM ou SMARTPHONE, ou sleep sur des ressources non verrouillées. |

## Exemple d'arguments

Arguments illustratifs qu'un agent fournit lors de l'appel de cet outil :

## Description

```text
This is the recommended way to run tests — it blocks until completion and returns per-test results. For very long campaigns (>10 minutes), use start_execution + poll_execution instead.

For ad-hoc runs: build_campaign(test_ids=…) → run_campaign. For a saved Tests campaign: build_campaign(test_campaign_id=…) → run_campaign(overrides=…).
```
