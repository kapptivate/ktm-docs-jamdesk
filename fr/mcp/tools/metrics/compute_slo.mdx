Calculer les métriques SLI/SLO à partir des résultats de requêtes InfluxDB.

<Note>
  **Lecture seule · Appelle des systèmes externes.** Appel sûr : cet outil ne modifie pas les données.
</Note>

## Paramètres

| Champ | Type | Requis | Description |
| --- | --- | --- | --- |
| `good_field` | string | Non | Nom du champ pour les événements réussis (par défaut 'good') |
| `operator` | string | Oui | Slug de l'opérateur (= nom de la base InfluxDB) |
| `precision` | string | Non | Précision temporelle de la requête (s, ms, u, ns) |
| `query` | string | Oui | Requête InfluxQL renvoyant les champs good/total |
| `slo_target` | number | Oui | Objectif SLO en pourcentage (par ex. 99.9) |

## Description

```text
Utilise la méthodologie de burn-rate de Google SRE.

Prérequis : exécutez query_metrics avec SHOW MEASUREMENTS et SHOW FIELD KEYS d'abord pour découvrir quelle measurement contient les compteurs de succès/échec.

Exemple : SELECT sum("number_success") AS good, sum("number_success") + sum("number_failed") AS total FROM "test" WHERE time > now() - 30d GROUP BY time(1d)

L'outil extrait les champs good/total, calcule le SLI, le budget d'erreur, le burn rate et la sévérité.

Seuils de sévérité (Google SRE) :
- critical : burn rate > 14.4x (pager immédiatement)
- red : burn rate > 6x (créer un ticket)
- yellow : burn rate > 1x (SLO en danger)
- green : burn rate <= 1x (dans le budget)
```
