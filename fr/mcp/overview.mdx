Le serveur MCP de Kapptivate offre à vos assistants IA un accès direct et contrôlé à votre plateforme. Configurez un client MCP avec une seule URL, authentifiez-vous avec votre clé API, et le modèle peut découvrir des [tests](/fr/tests/overview), exécuter des [campaigns](/fr/executions/run-campaign), interroger des métriques, gérer des [monitors](/fr/monitoring/overview) et traiter des [incidents](/fr/monitoring/incidents) grâce à des outils qui correspondent un à un aux fonctionnalités de Kapptivate.

Utilisez-le pour transformer une intention en action : "exécute la régression du checkout sur staging et résume les échecs", "crée un monitor pour le test de login toutes les cinq minutes", ou "affiche le taux de succès de la semaine dernière pour le produit paiements".

## Fonctionnement

Le serveur parle le [Model Context Protocol](https://modelcontextprotocol.io), le standard ouvert pour connecter des assistants à des outils. Kapptivate l'héberge sur un seul endpoint :

```
https://mcp.kapptivate.com/mcp
```

Votre client authentifie chaque requête avec l'en-tête `X-Kapptivate-API-Key`. La clé porte les mêmes permissions que votre compte Kapptivate, donc un agent ne peut faire que ce que vous pouvez faire.

<Warning>
  Votre clé API accorde le même accès que votre compte. Traitez-la comme un mot de passe : ne la commitez jamais dans un dépôt et ne la collez pas dans un chat partagé.
</Warning>
