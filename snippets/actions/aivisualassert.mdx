Use AI to validate that the current page state matches a natural language description. Capture a screenshot and have AI evaluate whether the visual state aligns with your assertion. This enables flexible, human-readable visual validation without brittle pixel-perfect matching.

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support visual assertions equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| assertion | Template | Yes | Free-text assertion describing expected visual state (e.g., "checkout button is green and says 'Pay Now'"). |
| mocked_response | Object | No | Mock response for testing (JSON). Use in development/testing only. |

## Example

```json
{
  "type": "aivisualassert",
  "version": "v1.0.0",
  "inputs": {
    "assertion": "the form is complete with all required fields filled"
  }
}
```

Validate button state:

```json
{
  "type": "aivisualassert",
  "inputs": {
    "assertion": "the submit button is blue and enabled"
  }
}
```

## Notes

- Captures current page screenshot and evaluates against assertion text.
- Assertion is free-form natural language; describe what should be visible.
- Step fails if AI determines assertion is not met.
- More flexible than selector-based checks but less deterministic.
- Common uses: verify form completeness, validate button states, check layout changes, QA visual regressions.
