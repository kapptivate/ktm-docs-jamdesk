Use AI to validate spelling and grammar on the current page for a specified language. The step captures all visible text and checks for spelling and grammar issues. Useful for content validation, localization testing, and quality assurance of user-facing text.

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support spell checking equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| language | String | Yes | Language code for spell check (e.g., `en` for English, `fr` for French). |
| mocked_response | Object | No | Mock response for testing (JSON). Use in development/testing only. |

## Example

```json
{
  "type": "aispellcheck",
  "version": "v1.0.0",
  "inputs": {
    "language": "en"
  }
}
```

Spell check for French:

```json
{
  "type": "aispellcheck",
  "inputs": {
    "language": "fr"
  }
}
```

## Notes

- Captures all visible text on the current page.
- Returns list of spelling and grammar issues if found.
- Step fails if issues are detected (configurable per step check).
- Useful for multi-language testing, content validation, and localization QA.
- Common uses: verify marketing content has no typos, validate translated pages, QA user-generated content.
