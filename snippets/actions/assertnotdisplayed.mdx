Assert that an element is NOT visible or does not exist in the DOM. The step fails if the element is found and visible. Use this for negative assertions, such as verifying that error messages or loading indicators have disappeared.

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support element absence assertions equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| locator | Object | Yes | CSS selector, XPath, or AI-driven description to locate the element. |
| timeout | Template (ms) | No | Max time to wait for element to be hidden. Default ~10 seconds. |
| wait_for | Object | No | Advanced wait condition (clickable, displayed, enabled, exist). |

Locator object fields:

| Field | Type | Description |
|-------|------|-------------|
| selector | String | CSS selector or XPath expression. |
| description | String | Human-readable description (for AI-driven locators). |
| enable_ai | Boolean | Use AI-assisted locator matching. |

## Example

```json
{
  "type": "assertnotdisplayed",
  "version": "v1.0.0",
  "inputs": {
    "locator": {
      "selector": ".error-banner"
    },
    "timeout": "5000"
  }
}
```

Wait for element absence:

```json
{
  "type": "assertnotdisplayed",
  "inputs": {
    "locator": {
      "selector": ".loading-spinner"
    },
    "wait_for": {
      "exist": {
        "timeout": 10000,
        "reverse": true
      }
    }
  }
}
```

## Notes

- Fails the step if element is visible within timeout.
- Passes if element is absent or hidden.
- Use `wait_for.exist.reverse: true` to wait for element disappearance.
- Common uses: verify error messages are gone, confirm loading indicators disappear, check that modal closes.
