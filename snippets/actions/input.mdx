Type text into an input field or textarea element. The text value supports templating, allowing you to use variables, globals, and template functions like `NOW()` or `UUID()`. The field is automatically cleared before new text is entered.

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support text input equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| locator | Object | Yes | CSS selector, XPath, or AI-driven description to locate the input field. |
| value | Template | Yes | Text to enter. Supports templating (e.g., `${USER_EMAIL}`). |
| wait_for | Object | No | Condition to wait for before or after input (clickable, displayed, enabled, exist). |
| timeout | Template (ms) | No | Max time to wait for field. Default ~10 seconds. |

Locator object fields:

| Field | Type | Description |
|-------|------|-------------|
| selector | String | CSS selector or XPath expression. |
| description | String | Human-readable description (for AI-driven locators). |
| enable_ai | Boolean | Use AI-assisted locator matching. |

## Example

```json
{
  "type": "input",
  "version": "v1.0.0",
  "inputs": {
    "locator": {
      "selector": "#email-field"
    },
    "value": "${ADMIN_EMAIL}"
  }
}
```

With AI-driven locator:

```json
{
  "type": "input",
  "inputs": {
    "locator": {
      "description": "the email input field"
    },
    "value": "user@example.com"
  }
}
```

## Notes

- Field is cleared before new text is entered.
- Value supports templating: test variables (`${var_name}`), globals (`${GLOBAL}`), and functions (`${UUID()}`, `${NOW()}`).
- Use [keys](/tests/actions/web/keys) to press Enter or Tab after input.
- Common uses: fill login forms, enter search queries, populate registration forms.
