Assert that an element is displayed (visible) on the page. The step fails if the element is not found or not visible within the timeout. Use this to verify page state before proceeding with subsequent actions.

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support element visibility assertions equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| locator | Object | Yes | CSS selector, XPath, or AI-driven description to locate the element. |
| timeout | Template (ms) | No | Max time to wait for element visibility. Default ~10 seconds. |
| wait_for | Object | No | Advanced wait condition (clickable, displayed, enabled, exist). |

Locator object fields:

| Field | Type | Description |
|-------|------|-------------|
| selector | String | CSS selector or XPath expression. |
| description | String | Human-readable description (for AI-driven locators). |
| enable_ai | Boolean | Use AI-assisted locator matching. |

## Example

```json
{
  "type": "assertdisplayed",
  "version": "v1.0.0",
  "inputs": {
    "locator": {
      "selector": ".success-message"
    },
    "timeout": "5000"
  }
}
```

With AI-driven locator:

```json
{
  "type": "assertdisplayed",
  "inputs": {
    "locator": {
      "description": "the green success banner at the top"
    },
    "wait_for": {
      "displayed": {
        "timeout": 10000
      }
    }
  }
}
```

## Notes

- Fails the step if element is not visible within timeout.
- Passes immediately if element is visible.
- Use [assert not displayed](/tests/actions/web/assertnotdisplayed) for negative assertions.
- Common uses: verify page navigation, confirm form submission success, check error messages.
