Check or uncheck a checkbox element. The action sets the checkbox to the desired state (checked or unchecked) regardless of its current state, making it idempotent. If the state actually changes, a `change` event is emitted.

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support checkbox control equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| locator | Object | Yes | CSS selector, XPath, or AI-driven description to locate the checkbox. |
| checked | Boolean | Yes | Desired state: `true` (check), `false` (uncheck). |
| wait_for | Object | No | Condition to wait for before or after toggle (clickable, displayed, enabled, exist). |
| timeout | Template (ms) | No | Max time to wait for checkbox. Default ~10 seconds. |

Locator object fields:

| Field | Type | Description |
|-------|------|-------------|
| selector | String | CSS selector or XPath expression. |
| description | String | Human-readable description (for AI-driven locators). |
| enable_ai | Boolean | Use AI-assisted locator matching. |

## Example

```json
{
  "type": "checkbox",
  "version": "v1.0.0",
  "inputs": {
    "locator": {
      "selector": "#terms-checkbox"
    },
    "checked": true
  }
}
```

Uncheck a checkbox:

```json
{
  "type": "checkbox",
  "inputs": {
    "locator": {
      "description": "the newsletter subscription checkbox"
    },
    "checked": false
  }
}
```

## Notes

- Idempotent: setting to the same state multiple times is safe.
- If state changes, a `change` event is triggered on the checkbox.
- Use with form-based UIs; some custom toggles may require [click](/tests/actions/web/click) instead.
- Common uses: accept terms and conditions, enable/disable features, manage form preferences.
