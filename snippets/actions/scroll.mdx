Scroll a page or scrollable container to bring an element into view. Supports smooth and instant scrolling, and allows control over vertical and horizontal alignment (start, center, end).

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support scrolling equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| locator | Object | No | CSS selector, XPath, or AI-driven description. If omitted, scrolls the window/body. |
| behavior | String | No | Scroll behavior: `smooth` or `auto` (instant). Default: `auto`. |
| block | String | No | Vertical alignment: `start`, `center`, `end`. Default: `start`. |
| inline | String | No | Horizontal alignment: `start`, `center`, `end`. Default: `start`. |

Locator object fields:

| Field | Type | Description |
|-------|------|-------------|
| selector | String | CSS selector or XPath expression. |
| description | String | Human-readable description (for AI-driven locators). |
| enable_ai | Boolean | Use AI-assisted locator matching. |

## Example

Scroll to element using smooth behavior:

```json
{
  "type": "scroll",
  "version": "v1.0.0",
  "inputs": {
    "locator": {
      "selector": "#checkout-button"
    },
    "behavior": "smooth",
    "block": "center"
  }
}
```

Scroll window to top:

```json
{
  "type": "scroll",
  "inputs": {
    "block": "start"
  }
}
```

## Notes

- If no locator is provided, scrolls the window or body element.
- `block` controls vertical alignment: `start` (top of view), `center` (middle), `end` (bottom).
- `inline` controls horizontal alignment (rarely needed).
- `smooth` scroll may take time; use [pause](/tests/actions/shared/pause) if needed to wait for animation.
- Common uses: bring element into view before click, scroll to bottom to load lazy content, navigate long pages.
