Extract the text content (`.textContent`) from an element and store it in a test variable for later use. This is useful for verifying dynamic content, capturing user-facing text, or using extracted values in subsequent steps.

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support text extraction equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| locator | Object | Yes | CSS selector, XPath, or AI-driven description to locate the element. |
| variable | String | No | Variable name to store extracted text. If omitted, text is only logged. |
| wait_for | Object | No | Pre-extraction wait condition (clickable, displayed, enabled, exist). |
| timeout | Template (ms) | No | Max time to wait for element. Default ~10 seconds. |

Locator object fields:

| Field | Type | Description |
|-------|------|-------------|
| selector | String | CSS selector or XPath expression. |
| description | String | Human-readable description (for AI-driven locators). |
| enable_ai | Boolean | Use AI-assisted locator matching. |

## Example

```json
{
  "type": "gettext",
  "version": "v1.0.0",
  "inputs": {
    "locator": {
      "selector": ".order-id"
    },
    "variable": "order_id"
  }
}
```

Extract from header and use in later steps:

```json
{
  "type": "gettext",
  "inputs": {
    "locator": {
      "description": "the user name in the header"
    },
    "variable": "logged_in_user"
  }
}
```

## Notes

- Reads `.textContent` of the element (all text, including nested elements).
- Extraction can be combined with [setvariable](/tests/actions/shared/setvariable) for post-processing.
- Use [getvalue](/tests/actions/web/getvalue) for form field values instead.
- Common uses: capture order IDs, verify message text, extract user information.
