Extract either text content or form value from an element, with automatic fallback. First tries to read text; if no text is available, falls back to the element's value property. Use this when you're unsure whether a field is text-based or a form input.

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support fallback extraction equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| locator | Object | Yes | CSS selector, XPath, or AI-driven description to locate the element. |
| variable | String | No | Variable name to store extracted text or value. If omitted, result is only logged. |
| wait_for | Object | No | Pre-extraction wait condition (clickable, displayed, enabled, exist). |
| timeout | Template (ms) | No | Max time to wait for element. Default ~10 seconds. |

Locator object fields:

| Field | Type | Description |
|-------|------|-------------|
| selector | String | CSS selector or XPath expression. |
| description | String | Human-readable description (for AI-driven locators). |
| enable_ai | Boolean | Use AI-assisted locator matching. |

## Example

```json
{
  "type": "gettextorvalue",
  "version": "v1.0.0",
  "inputs": {
    "locator": {
      "selector": ".display-field"
    },
    "variable": "field_content"
  }
}
```

## Notes

- Tries text extraction first (`.textContent`), then falls back to value (`.value`).
- Useful for mixed UI components that might be text labels or input fields.
- If both text and value are empty, an empty string is stored.
- Common uses: extract from custom form fields, handle variable UI components.
