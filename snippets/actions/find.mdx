Locate an element using a selector or AI-driven description and wait for specified conditions (clickable, displayed, enabled, exist). Unlike other actions, find does not interact with or extract data from the element. It only verifies that the element is present and in the desired state.

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support element location equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| locator | Object | Yes | CSS selector, XPath, or AI-driven description to locate the element. |
| wait_for | Object | No | Conditions to wait for (clickable, displayed, enabled, exist). |
| timeout | Template (ms) | No | Max time to wait for element. Default ~10 seconds. |

Locator object fields:

| Field | Type | Description |
|-------|------|-------------|
| selector | String | CSS selector or XPath expression. |
| description | String | Human-readable description (for AI-driven locators). |
| enable_ai | Boolean | Use AI-assisted locator matching. |

## Example

```json
{
  "type": "find",
  "version": "v1.0.0",
  "inputs": {
    "locator": {
      "selector": "#checkout-button"
    },
    "wait_for": {
      "clickable": {
        "timeout": 5000
      }
    }
  }
}
```

## Notes

- Find does not interact with the element. Use [click](/tests/actions/web/click) or [tap](/tests/actions/smartphone/tap) to interact.
- Useful for element healing in the next step (prefind) or for verification before complex flows.
- If element is not found within timeout, the step fails.
- Common uses: verify element exists before click, wait for UI component to be ready, validate dynamic content loads.
