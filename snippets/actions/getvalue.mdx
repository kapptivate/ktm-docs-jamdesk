Extract the value (`.value` property) from a form input, textarea, or select element and store it in a test variable. Use this to verify form data, capture user-entered values, or validate field state.

## Channels

- ✅ Web
- ✅ Smartphone (Android/iOS)
- ✅ Cellular

All channels support form value extraction equally.

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| locator | Object | Yes | CSS selector, XPath, or AI-driven description to locate the form field. |
| variable | String | No | Variable name to store extracted value. If omitted, value is only logged. |
| wait_for | Object | No | Pre-extraction wait condition (clickable, displayed, enabled, exist). |
| timeout | Template (ms) | No | Max time to wait for field. Default ~10 seconds. |

Locator object fields:

| Field | Type | Description |
|-------|------|-------------|
| selector | String | CSS selector or XPath expression. |
| description | String | Human-readable description (for AI-driven locators). |
| enable_ai | Boolean | Use AI-assisted locator matching. |

## Example

```json
{
  "type": "getvalue",
  "version": "v1.0.0",
  "inputs": {
    "locator": {
      "selector": "#email-input"
    },
    "variable": "form_email"
  }
}
```

Extract from search field:

```json
{
  "type": "getvalue",
  "inputs": {
    "locator": {
      "description": "the search input field"
    },
    "variable": "search_query"
  }
}
```

## Notes

- Reads `.value` property of form elements (input, textarea, select).
- Use [gettext](/tests/actions/web/gettext) for element text content instead.
- Extracted value can be used in assertions or passed to subsequent steps.
- Common uses: verify form is pre-filled, capture user inputs, validate dropdown selections.
