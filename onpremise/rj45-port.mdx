title: Which RJ45 port is configured on your robot?
description: Find which Ethernet port carries the static IP, on every robot generation.
sidebarTitle: Identify the RJ45 port
---

When a robot has a static IP configured, you need to know which of its Ethernet ports that configuration applies to. The method depends on the hardware generation.

## Mac mini robots

A single Ethernet port, nothing to identify.

## Latest Linux robots (graphical interface)

The port numbering and the matching `lan1`/`lan2`/`lan3` connections are covered in [Install a robot (software)](/onpremise/install-software).

## Old Linux robots on Ubuntu (no graphical interface)

Log in with your credentials. Ubuntu greets you with a full prompt:

<Frame caption="Ubuntu login screen">
  ![Ubuntu prompt](/images/onpremise/rj45-port-02.webp)
</Frame>

List the interfaces:

```bash
ifconfig -s
```

<Frame caption="Spot the active interface">
  ![ifconfig output](/images/onpremise/rj45-port-03.webp)
</Frame>

Look at the rows starting with `enp` and spot the one whose RX/TX values are not 0: that interface is the one physically connected to the RJ45 cable.

Then find the netplan file and read the configuration:

```bash
ls /etc/netplan
vim /etc/netplan/01-netcfg.yaml
```

<Frame caption="The netplan file and its content">
  ![netplan directory](/images/onpremise/rj45-port-04.webp)
  ![netplan configuration](/images/onpremise/rj45-port-05.webp)
</Frame>

The interface configured with a static IP is the one with `dhcp` set to `no` (exit vim with `:q!`). If it differs from the active interface you found with `ifconfig`, move the cable to another RJ45 port and check again.

## Old Linux robots on Fedora (no graphical interface)

Fedora shows a minimal prompt after login:

<Frame caption="Fedora login prompt">
  ![Fedora prompt](/images/onpremise/rj45-port-06.webp)
</Frame>

List the interfaces the same way:

```bash
ifconfig -s
```

<Frame caption="Spot the active interface">
  ![ifconfig output](/images/onpremise/rj45-port-07.webp)
</Frame>

Spot the `enp` interface with non-zero RX/TX, then read its configuration:

```bash
nmcli device show
```

<Frame caption="The interface configuration: IP, gateway, DNS">
  ![nmcli output](/images/onpremise/rj45-port-08.webp)
</Frame>

Scroll to your `enp` interface: the IP, gateway, and DNS configuration is there. As on Ubuntu, if the statically configured interface is not the active one, move the cable to another port and check again.
