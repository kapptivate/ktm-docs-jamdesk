API tests hit HTTP services directly, useful for backend checks and end-to-end flows that mix UI and API calls. They run on the Web agent. Here's how to make one.

<Steps>
  <Step>
    **Create the test.** Open the **Tests** section for your product, click **Create**, and choose the **API** channel. Name the test.
  </Step>
  <Step>
    **Configure the request.** Set the HTTP method (GET, POST, PUT, PATCH, DELETE) and the URL, then add headers and a body if the request needs them.
  </Step>
  <Step>
    **Assert the response.** Check the status code, assert on the body or headers, and extract values from the response for use in later steps.
  </Step>
  <Step>
    **Run and read the result.** Click **Run**, then open the execution under **Executions** for status and a per-request timing breakdown (DNS, TCP, TLS, transfer).
  </Step>
</Steps>
